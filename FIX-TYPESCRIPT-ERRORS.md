## CAT-1 — TanStack Query v5: `onSuccess` removed from `useQuery` options

**Root cause:** TanStack Query v5 removed `onSuccess` from `useQuery`. It only exists on `useMutation`. Using it causes TypeScript overload resolution to fail, which makes the return type of `useQuery` collapse to `UseQueryResult<unknown>` — this is the root cause of ALL `TabDrawer/index.tsx` errors about `unknown` and `{}`.

**Files to fix:**

- `src/entities/product/model/queries.ts` — 3 occurrences (lines ~62, ~99, ~134)
- `src/entities/tab/model/queries.ts` — 1 occurrence (line ~77)

**Fix pattern:** Remove `onSuccess` from the `useQuery` options object. Move the store-sync call into a `useEffect` inside the same hook function, watching the query's `data`.

```ts
// BEFORE (broken):
export function useProducts() {
  const setProducts = useProductStore(state => state.setProducts);
  return useQuery({
    queryKey: ['products'],
    queryFn: async () => {
      /* ... */ return products;
    },
    staleTime: 5 * 60 * 1000,
    onSuccess: products => {
      // ← REMOVE THIS BLOCK
      setProducts(products);
    },
  });
}

// AFTER (correct):
import { useEffect } from 'react';

export function useProducts() {
  const setProducts = useProductStore(state => state.setProducts);
  const query = useQuery({
    queryKey: ['products'],
    queryFn: async () => {
      /* ... */ return products;
    },
    staleTime: 5 * 60 * 1000,
    // onSuccess is gone — removed in TanStack Query v5
  });
  useEffect(() => {
    if (query.data) setProducts(query.data);
  }, [query.data, setProducts]);
  return query;
}
```

Apply this pattern to ALL three hooks in `product/model/queries.ts` (`useProducts`, `useCategories`, `useModifiers`) and `useTabs` in `tab/model/queries.ts`.

**Side effect:** Once CAT-1 is fixed, the `unknown` type errors in `src/widgets/TabDrawer/index.tsx` (lines 70–106) will resolve automatically — do NOT separately edit `TabDrawer/index.tsx` for those errors.

## CAT-20 — `src/entities/staff/model/store.ts`: `exactOptionalPropertyTypes` with Zustand `set`

**Root cause (lines ~112, ~154):** The Zustand `set` function is called with a function returning a union of partial types. With `exactOptionalPropertyTypes: true`, TypeScript requires that partial updates cannot contain keys with value `never` or union types that are too broad.

**Fix:** Replace the conditional object-spread pattern with explicit conditional `set` calls:

Instead of:

```ts
set(state => ({
  ...state,
  ...(condition ? { currentStaff: staff } : {}),
}));
```

Use:

```ts
if (condition) {
  set({ currentStaff: staff });
} else {
  set({ allStaff: newList });
}
```

This avoids the union return type that confuses `exactOptionalPropertyTypes`. Apply to all failing `set(...)` calls in the store.
