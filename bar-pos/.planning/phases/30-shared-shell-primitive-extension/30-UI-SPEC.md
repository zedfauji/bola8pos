## Component Contract: `PageContainer`

### Prop signature (extend in place — do not create a new component)

```ts
export interface PageContainerProps {
  children: React.ReactNode;
  title: string;
  description?: string;
  actions?: React.ReactNode;
  className?: string;
  /** NEW — route to navigate back to. Omit to render no back button (e.g. top-level sections). */
  backTo?: string;
  /** NEW — back button label. Defaults to "Home" when omitted, matching current BackToHomeButton copy. */
  backLabel?: string;
}
```

- `backTo` is optional. When absent, `SectionHeader` renders exactly as it does today (no back button) — this is required for any future non-exempt page that has no sensible parent (there are none in this phase's 14-caller migration, but the prop must not be mandatory).
- `backLabel` defaults to `'Home'` when `backTo` is provided but `backLabel` is not — this preserves the exact copy every current `BackToHomeButton` caller already shows.
- `exactOptionalPropertyTypes` is on (project-wide TS setting) — write `backTo?: string` (not `backTo: string | undefined`) here because these are *component props* with a sensible "prop absent" default, not mutation inputs. Do not spread `undefined` into `SectionHeader`; use the same conditional-spread pattern `PageContainer` already uses for `description`/`actions` (`{...(backTo && { backTo })}`).

### Render placement — inline in `SectionHeader`, not a new sibling

Current (deleted) pattern — a separate full-width strip *above* `PageContainer`/`SectionHeader`:

```tsx
// BEFORE — src/pages/audit/index.tsx and 13 other callers
<div className="px-4 pt-4">
  <Button variant="ghost" size="sm" asChild>
    <Link to="/home"><ChevronLeft className="mr-1 h-4 w-4" />Home</Link>
  </Button>
</div>
<PageContainer title="Audit Log">…</PageContainer>
```

New pattern — back button renders **inside** `SectionHeader`'s left column, **directly above the `<h2>` title**, inside the existing `space-y-1` block:

```tsx
// AFTER — SectionHeader.tsx, left column only, all other SectionHeader markup unchanged
<div className="space-y-1">
  {backTo && (
    <Button variant="ghost" size="sm" asChild className="-ml-2.5 h-6 px-2 text-xs">
      <Link to={backTo}>
        <ChevronLeft className="mr-1 h-3.5 w-3.5" />
        {backLabel ?? 'Home'}
      </Link>
    </Button>
  )}
  <div className="flex items-center gap-2">
    <h2 className="text-2xl font-bold tracking-tight">{title}</h2>
    {badge !== undefined && <Badge variant="secondary" aria-label={`Count: ${String(badge)}`}>{badge}</Badge>}
  </div>
  {description && <p className="text-sm text-muted-foreground">{description}</p>}
</div>
```

Rationale for the size/margin adjustment (`h-6 px-2 text-xs` instead of the old `size="sm"` `h-7`, plus `-ml-2.5`): the old button lived in its own `px-4 pt-4` strip with nothing else next to it, so its default `sm` padding was correct there. Placed directly above an `text-2xl` title inside `SectionHeader`'s tight `space-y-1` stack, the unmodified `sm` size reads as oversized and its left padding visually mis-aligns with the title's left edge. `-ml-2.5` cancels the button's own `px-2.5`/`px-2` horizontal padding so the **ChevronLeft icon's left edge lines up with the title's left edge** — this is the one intentional visual micro-adjustment in this phase, made solely to correct a placement side-effect of moving the button, not a redesign. It is still the same `ghost` variant, same `ChevronLeft` icon, same `Link`/`asChild` composition — no new visual language.

**This is the only markup change requiring judgment. Everything else is a mechanical prop-add + 14 call-site edits (replace `<BackToHomeButton />` + wrapper div with `backTo="..."` prop on the page's own `<PageContainer>` call).**

## Spacing Scale

Declared values (existing project scale — 8-point, already in use, no additions):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | `ChevronLeft` icon-to-label gap (`mr-1`) |
| sm | 8px | `SectionHeader`'s internal `space-y-1` micro-stack (title/back-button/description) |
| md | 16px | `SectionHeader`'s `gap-4` between text column and actions slot |
| lg | 24px | `PageContainer`'s `p-6` outer padding, `space-y-6` between header and content |

Exceptions: **one** — the back button's `-ml-2.5` negative margin (documented above, cancels the button's own horizontal padding to align icon-left with title-left). This is a documented alignment correction, not an arbitrary-value violation of TOKEN-02 (Phase 31 concern); it uses the existing `2.5` step already present in the button's own CVA padding scale (`px-2.5`), not an invented arbitrary pixel value.
