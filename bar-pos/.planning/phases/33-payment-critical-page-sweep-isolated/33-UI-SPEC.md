# Phase 33 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Phase type: mechanical rollout, not new design.** This phase introduces zero new visual patterns, zero new colors, zero new copy, zero new layouts. Every token/component/pattern below already shipped in Phases 30-32. This UI-SPEC formalizes the *existing* `POSButton`/`Button`/`ConfirmDialog` contracts into a per-file component inventory so the planner and executor have one unambiguous target per button. **Zero prop/handler/validation behavior change is the hard constraint (Success Criterion 1, COMPONENT-04)** — every row below is a `className`/prop-only diff; no `onClick`, `disabled`, `aria-*`, or business-logic expression may change.

### Variant Decisions Required (D-03b pitfall — must not silently regress visuals)

Every raw `<button>` with **no** prior CVA styling (transparent/text-only) must receive an **explicit** `variant="ghost"` when converted — omitting `variant` defaults `POSButton`/`Button` to `variant="default"` (`bg-primary`, solid background), which is a real, unauthorized visual change. This applies to:
- `pos/index.tsx` panel toggle
- `CartPanel.tsx` Clear Cart
- `PaymentForm.tsx` Reset-to-computed

`RefundSheet.tsx`'s two buttons and `TabPaymentCard.tsx`'s card have no variant ambiguity (already `variant="outline"` or implicitly `variant="default"` matching the raw button's actual solid-card appearance) — confirm 1:1 before/after screenshot parity per executor discretion, no variant change needed there.

### Sizing Fix Required (D-03a pitfall — icon-only buttons)

`touchSize`/`min-h-[Npx]` only sets height. Two icon-only buttons need an explicit width companion or they will render as tall, narrow slivers instead of squares:
- `pos/index.tsx` panel toggle: target 44×44px — pair `touchSize="default"` with `w-11` (44px) explicitly, replacing the old fixed `h-9 w-9`.
- `SplitTabSheet.tsx` Remove check: target 72×72px — drop `size="icon-sm"` (which sets both `size-7`/28px dimensions) and pair `touchSize="xl"` with `className="w-[72px]"` (or accept the icon renders with default `size="default"` horizontal padding if a full 72×72 square reads as oversized for a remove-row icon — executor may choose the smaller of the two options but must document which was chosen in the commit body).

### Focus Ring Contract (FOCUS-02)

`focusEmphasis="high"` (→ `focus-visible:ring-4 focus-visible:ring-ring`) applies to exactly these 6 targets — all D-03 (72px) tier, sizing and ring stay in lockstep per D-05:
1. Process Payment / Process split payment — `PaymentForm.tsx` ~L1000
2. Remove payment leg — `PaymentForm.tsx` ~L721
3. Request approval (refund confirm) — `RefundSheet.tsx` ~L314
4. Void order confirm — `VoidOrderDialog.tsx` (via `confirmClassName`)
5. Confirm Split — `SplitTabSheet.tsx` ~L792
6. Remove check — `SplitTabSheet.tsx` ~L737

Everything else in this phase's scope stays at `focusEmphasis="default"` (the CVA base's `focus-visible:ring-3 focus-visible:ring-ring/50`, unchanged).
