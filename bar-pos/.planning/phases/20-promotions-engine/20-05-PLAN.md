<objective>
Deliver both locked pool-time promotion behaviors (D-05): (a) a direct pool-billing discount applied when a session stops, and (b) purchase-triggered bonus pool minutes consumed against a session's charge. This requires introducing the FIRST server-side RPC boundary for the pool-session stop write (Pitfall 3 — today it is a raw client `.update()`), porting the pool billing math into PL/pgSQL so the discount + grant consumption + charge write are atomic.

Purpose: Make pool billing server-authoritative for promotions (mirroring how `close_tab` wrapped a previously-raw tab write in Phase 14), and wire the two distinct pool evaluation paths onto the shared `promotions` schema per D-05.
Output: Two additive migrations (evaluate v2 pool_grant branch + `stop_pool_session` RPC) + a live-integration scaffold. The client rewire that CALLS `stop_pool_session` is Plan 20-08; this plan only authors the server side.
</objective>

    Create migration `20260710000006_stop_pool_session_rpc.sql` defining `stop_pool_session(p_session_id uuid, p_expected_version int DEFAULT NULL) RETURNS jsonb LANGUAGE plpgsql SECURITY DEFINER`. Body, all in one transaction:
    1. `SELECT ... FROM pool_sessions WHERE id = p_session_id FOR UPDATE` capturing `started_at`, `tab_id`, `table_id`, current `version`. Raise if not found. Version guard: `IF p_expected_version IS NOT NULL AND v_current_version IS DISTINCT FROM p_expected_version THEN RAISE EXCEPTION ... USING ERRCODE = 'P0V01'` (mirror close_tab; STALE_VERSION contract).
    2. Read `rate_per_hour` from `pool_tables WHERE id = v_table_id` (server-authoritative rate — do NOT trust a client-passed rate). Read `firstHourMode` from `settings WHERE key = 'billing'` (`value->>'firstHourMode'`, default 'prorated').
    3. Consume grants: `SELECT COALESCE(SUM(pool_minutes_granted),0) INTO v_prepaid FROM applied_promotions WHERE tab_id = v_tab_id AND pool_minutes_granted IS NOT NULL AND consumed_at IS NULL` (only when v_tab_id IS NOT NULL). Then `UPDATE applied_promotions SET consumed_at = now(), pool_session_id = p_session_id WHERE tab_id = v_tab_id AND pool_minutes_granted IS NOT NULL AND consumed_at IS NULL`.
    4. Port computePoolSessionBilling math server-side EXACTLY: `v_elapsed_minutes := EXTRACT(EPOCH FROM (now() - v_started_at)) / 60`; if `firstHourMode = 'full' AND v_elapsed_minutes < 60` then `v_base_billed := 60` else `v_base_billed := v_elapsed_minutes`; `v_chargeable := GREATEST(0, v_base_billed - v_prepaid)`; `v_billed_minutes := CASE WHEN v_chargeable = 0 THEN 0 ELSE CEIL(v_chargeable / 15.0) * 15 END`; `v_base_charge := ROUND((v_billed_minutes / 60.0) * v_rate_per_hour, 2)`.
    5. Apply pool_billing promotions: `v_total := v_base_charge`; loop `FOR v_promo IN SELECT * FROM promotions p WHERE p.is_active AND p.target_type = 'pool_billing' AND is_promotion_available(p.id, now()) ORDER BY p.priority ASC, p.created_at ASC, p.id ASC` with the same compounding CASE + `GREATEST(0, ROUND(...,2))` clamp as evaluate_promotions_for_item (percentage/fixed_amount compound, fixed_price resets). Per applied promo, `INSERT INTO applied_promotions (promotion_id, promotion_name_snapshot, target_type, discount_type, discount_value, tab_id, pool_session_id, original_amount, discounted_amount) VALUES (v_promo.id, v_promo.name, 'pool_billing', v_promo.discount_type, v_promo.discount_value, v_tab_id, p_session_id, v_original, v_total)`.
    6. `UPDATE pool_sessions SET stopped_at = now(), billed_minutes = v_billed_minutes, total_charge = v_total, version = v_current_version + 1 WHERE id = p_session_id`.
    7. `PERFORM record_audit('promotion.apply', 'pool_session', p_session_id, v_before, jsonb_build_object('billedMinutes', v_billed_minutes, 'baseCharge', v_base_charge, 'prepaidMinutes', v_prepaid, 'totalCharge', v_total), 'rpc', NULL)` (7-arg current signature).
    8. RETURN a jsonb of the updated session row (id, stopped_at, billed_minutes, total_charge, version, tab_id, table_id) so the client can map it. `GRANT EXECUTE ON FUNCTION public.stop_pool_session(uuid, int) TO authenticated`. Header comment documents that this RPC is now the sole authoritative writer of `pool_sessions.total_charge`/`billed_minutes`, replacing the raw client `.update()` (rewired in Plan 20-08), and that it consumes pool_grant minutes at stop time.
