**Step 2 — Touch-target fixes.**

Read each UI file. For any `<Button>` without a `size` prop that is a primary action trigger (seat, notify, cancel, no-show), change to `size="lg"`. If `POSButton` is exported from `@shared/ui`, use that instead — it already has the correct touch target.

Minimum change pattern:
```tsx
// Before
<Button onClick={handleSeat}>Seat Party</Button>

// After (if POSButton not available)
<Button size="lg" onClick={handleSeat}>Seat Party</Button>

// After (if POSButton is available in @shared/ui)
<POSButton onClick={handleSeat}>Seat Party</POSButton>
```

**Step 3 — Focus-trap verification in Sheet components.**

In `SeatWaitlistPartySheet.tsx` and `AddWaitlistEntryForm.tsx`, check if `SheetContent` has a focusable element on open. If the first interactive element (e.g. a party name input or confirm button) does not have `autoFocus`, add it:

```tsx
// On the first input or button inside SheetContent
<Input autoFocus ... />
// OR on the SheetContent itself if radix-ui supports ref forwarding:
// <SheetContent ref={sheetRef}> — less reliable; prefer autoFocus on first element
```

If focus-trap is already working (SheetContent uses Radix UI Dialog under the hood which traps focus by default), add a comment:
```tsx
{/* Focus is trapped by Radix UI SheetContent — no explicit autoFocus needed */}
```
Either a working focus trap comment OR an explicit `autoFocus` prop must be present so the grep AC passes.

**Step 4 — Error toast copy alignment.**

Read each mutation hook. If the error toast uses `'Something went wrong'` as the title with no feature context, update to a specific message:

```typescript

// After — useSeatWaitlistParty example
toast({ title: 'Could not seat party', description: 'Try again or refresh the waitlist.', variant: 'destructive' });
```

Only change toast strings that are currently generic. If a hook already has specific copy, leave it.
