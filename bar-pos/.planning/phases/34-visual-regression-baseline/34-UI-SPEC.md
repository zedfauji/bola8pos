# Phase 34 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Scope Note (read first)

**This phase introduces zero new UI.** It is test infrastructure only: an isolated Playwright visual-regression config (`playwright.visual.config.ts`) plus a screenshot-baseline spec (`e2e/visual/45-visual-baseline.spec.ts`) that photographs the UI already shipped and standardized across Phases 29–33.1 (drift audit, shared shell, component/token/spacing sweep, touch/focus sweep, payment-critical sweep). No new screens, layouts, copy, colors, spacing values, or component variants are created.

Consequently, five of the six normal design-contract dimensions (Spacing, Typography, Color, Copywriting, Registry) resolve to **N/A — no new UI in this phase** below, stated explicitly rather than padded with generic boilerplate. The one real design-adjacent decision this phase makes is which existing elements receive a `data-testid` attribute so the visual-regression spec can locate and `mask()` dynamic regions (a running timer, a live clock, a KDS board root, a toast container). Each addition is a pure test hook — it adds an HTML attribute only, with **zero change to existing className, layout, copy, or behavior**. This contract exists to make that "pure test hook, not a visual change" guarantee explicit and checkable.

Source: `.planning/phases/34-visual-regression-baseline/34-CONTEXT.md` (D-01..D-17), `.planning/phases/34-visual-regression-baseline/34-RESEARCH.md` (Wave 0 Gaps, Pitfalls 2–5).

## Test-Hook Contract (this phase's actual deliverable)

This section replaces the standard "new visual element" contract with the actual artifact this phase produces from a UI-perspective: two `data-testid` additions, both confirmed pure test hooks (no styling/behavior change) by direct inspection in RESEARCH.md.

| Element | File | Attribute to add | Why | Behavior/style change |
|---------|------|-------------------|-----|------------------------|
| Live wall-clock display | `src/shared/ui/LiveTimeDisplay.tsx` | `data-testid="live-time-display"` | Rendered in both `/kds` and `/kds-bar` page headers (`PageContainer`'s `actions` slot); a `setInterval`-driven clock with no existing selector — needed so the visual spec can `mask()` it on both KDS baselines (RESEARCH.md Pitfall 4) | None — attribute-only addition to the existing `<span className="text-muted-foreground text-sm tabular-nums">` |
| KDS board root | `src/widgets/KdsBoard/index.tsx` | `data-testid="kds-board"` on the root `<div className="grid gap-6 p-6...">` | Only needed if the KDS queue is non-empty at capture time (D-16: 1–2 pending orders seeded so card layout/spacing isn't hidden by an empty state) — masks the whole board region as one box since individual card relative-age labels (`"2 min"`) have no stable selector (RESEARCH.md Pitfall 3) | None — attribute-only addition to the existing grid container div |

**Already-existing testids to reuse — no new addition needed:**

| Element | Selector | Route(s) | Source |
|---------|----------|----------|--------|
| Pool session elapsed-minutes text | `data-testid="elapsed-minutes"` | `/pool-tables/:tableId` | Confirmed in `src/pages/pool-table-status/index.tsx` / `TableStatusPanel` |
| Happy-hour countdown banner | `data-testid="active-promotions-banner"` | `/pos` | Confirmed in `src/widgets/OrderPanel/HappyHourBanner.tsx` |
| Toast container | `data-sonner-toaster` | all routes (defensive mask per D-10) | Library-provided (sonner), not app code — confirmed via `node_modules/sonner/dist/index.mjs` |

**Explicitly out of scope for new `data-testid` additions in this phase** (per RESEARCH.md's laziest-option recommendation, confirm at implementation time rather than pre-emptively adding selectors):
- `/pool-tables` grid — capture the idle-state grid *before* seeding any occupied session (RESEARCH.md Anti-Pattern), sidestepping the need to mask `PoolTableCard`'s live minutes counter / SVG timer overlay entirely.
- `/staff` live shift-duration text — no masking selector added unless a clocked-in staff member is unavoidably present at capture time; prefer capturing with no active shift if seed state allows.

**Hard constraint carried into planning/execution:** any `data-testid` addition in this phase MUST NOT alter `className`, DOM structure (beyond the attribute itself), copy, or event handlers on the target element. A diff that changes anything else on these two files is out of contract and must be flagged by `gsd-ui-checker`.
