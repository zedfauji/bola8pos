| Property | Value | Source |

## Spacing Scale

The project uses the Tailwind 4px grid throughout. Recipe editor components must use only these values:

| Token | px | Use |
|-------|----|-----|
| `gap-1` / `space-y-1` | 4px | Tight label-to-input gap, hint text |
| `gap-2` / `space-y-2` | 8px | Label-to-input within a field group; icon + text; badge + label; row cell spacing |
| `gap-4` | 16px | Between fields in a form section |
| `gap-6` | 24px | Between major sections (e.g. recipe rows vs footer) |
| `p-4` | 16px | Tab content inner padding |
| `p-6` | 24px | Panel padding (depletion preview panel) |
| `pt-4` | 16px | Form footer top spacing (matches `ProductForm`) |
| `mt-auto` | auto | Push footer to bottom of scrollable form |
| `min-h-[44px]` | 44px | Minimum touch target height for recipe row remove button |

> **Exception:** The `IngredientAutocomplete` trigger button must be at least 44px tall to meet WCAG 2.1 AA touch target. Use `h-11` (44px) on the trigger.

> **Implementation note:** `IngredientForm.tsx` currently uses `space-y-1.5` (6px) — align to `gap-2` / `space-y-2` (8px) during implementation of this phase.

## Typography

Inherits the project's Geist Variable + shadcn scale. No new type styles are introduced.

| Role | Tailwind class | px equivalent | Weight | Line-height | Where used |
|------|---------------|--------------|--------|-------------|------------|
| Section header | `text-sm font-semibold` | 14px, 600 | semibold | 1.4 | "1 serving consumes…", "Selling 1 [product] will deplete:" |
| Body / label | `text-sm` | 14px, 400 | regular | 1.5 | Row labels, field labels, UOM display |
| Hint / muted | `text-xs text-muted-foreground` | 12px, 400 | regular | 1.5 | Hint text under fields, yield label |
| Error | `text-xs text-destructive` | 12px, 400 | regular | 1.5 | Inline field errors (matches `IngredientForm`) |
| Numeric input | `font-mono text-sm` | 14px | 400 | 1.5 | Qty and yield inputs (matches `IngredientForm` cost field) |
| Badge / UOM chip | `text-xs font-semibold` | 12px, 600 | semibold | 1 | UOM display badge in autocomplete and row |

**Rule:** Maximum 4 sizes in use (xs=12px, sm=14px, base=16px for inputs, mono-sm=14px). Maximum 2 weights (400 regular + 600 semibold). No new heading sizes. Weight 500 (medium) is not used anywhere in this phase.

## Color

All colors are CSS variable tokens from `globals.css`. No hardcoded hex or oklch values in component code.

### 60 / 30 / 10 Split

| Role | % | Token | What it covers |
|------|---|-------|----------------|
| Dominant surface | 60% | `bg-background` | Tab content area, row backgrounds |
| Secondary surface | 30% | `bg-card` / `bg-muted` | Depletion preview panel, popover/command background |
| Accent | 10% | `bg-primary` / `text-primary` | Save button, focused state ring |

### Semantic Colors — Reserved Uses

| Token | Semantic meaning | Exact elements it applies to |
|-------|-----------------|------------------------------|
| `text-destructive` / `border-destructive` | Error, out-of-stock warning | Inline field validation errors; recipe row UOM when current stock < implied qty; negative-stock toast border |
| `var(--pos-danger)` (oklch red) | Negative stock visual — danger | Ingredient row highlight when stock would go negative; audit ledger negative value text |
| `var(--pos-accent)` (oklch green) | Positive stock — safe | Ingredient row highlight when stock is sufficient; depletion preview "stock OK" indicator |
| `text-muted-foreground` | Placeholder, hint, read-only UOM label | UOM read-only labels in recipe rows; empty state body copy; hint text |
| `text-foreground` | Primary content | Ingredient name in autocomplete, qty values |
| `bg-popover` | Autocomplete dropdown surface | `IngredientAutocomplete` Command popover background |
| `border-border` | Standard borders | Recipe row container, tab section dividers |

### Stock Indicator Colors (depletion preview panel)

| State | Visual treatment | Token |
|-------|-----------------|-------|
| Sufficient stock | Row text in `text-foreground`, icon `text-pos-accent` | `var(--pos-accent)` |
| Low stock (current < implied qty, still > 0) | `text-yellow-500` + `AlertTriangle` icon | Tailwind `text-yellow-500` — one-time exception, see note |
| Out of stock / negative | `text-destructive` + `AlertTriangle` icon | `var(--destructive)` |

> **Note on yellow:** The project has no `--pos-warning` token. Use Tailwind `text-yellow-500` for the low-stock state as a pragmatic exception. If Phase 5 introduces a `--pos-warning` token, migrate this class then.

### New: `IngredientAutocomplete` (`src/shared/ui/IngredientAutocomplete/`)

**Justification for shared/ui placement:** Used by the Recipe Editor (Phase 4), will be reused in prep management (Phase S3c) and potentially in manual stock adjustments. Meets the 3+ feature uses threshold.

#### Anatomy

```
<Popover>
  <PopoverTrigger asChild>
    <Button role="combobox" aria-expanded={open} className="h-11 w-full justify-between">
      {selected ? "{name} · {uom badge}" : "Select ingredient…"}
      <ChevronsUpDown className="ml-2 h-4 w-4 opacity-50" />
    </Button>
  </PopoverTrigger>
  <PopoverContent className="w-[--radix-popover-trigger-width] p-0">
    <Command>
      <CommandInput placeholder="Search ingredients…" />
      <CommandEmpty>No ingredient found.</CommandEmpty>
      <CommandList>
        <CommandGroup>
          {ingredients.map(ing => (
            <CommandItem key={ing.id} value={ing.name}>
              <Check className={cn("mr-2 h-4 w-4", selected?.id === ing.id ? "opacity-100" : "opacity-0")} />
              {ing.name}
              <Badge variant="secondary" className="ml-auto text-xs">{ing.uom}</Badge>
            </CommandItem>
          ))}
        </CommandGroup>
      </CommandList>
    </Command>
  </PopoverContent>
</Popover>
```

#### Component Props

```typescript
interface IngredientAutocompleteProps {
  ingredients: Pick<Ingredient, 'id' | 'name' | 'uom'>[];
  value: string | null;           // ingredient id
  onChange: (id: string) => void;
  disabled?: boolean;
  placeholder?: string;           // default: "Select ingredient…"
  isLoading?: boolean;
}
```

#### States

| State | Visual |
|-------|--------|
| Default (no selection) | Trigger shows placeholder in `text-muted-foreground` |
| Selected | Trigger shows `{name} · {uom}` in `text-foreground`; check icon visible in list item |
| Open/focused | Popover open; `CommandInput` auto-focused; trigger has `ring-2 ring-ring` |
| Loading | Trigger disabled; `Skeleton` `h-11 w-full rounded-md` replaces trigger |
| Empty search | `CommandEmpty` shows "No ingredient found." in `text-muted-foreground text-sm text-center py-6` |
| Disabled | Trigger `disabled:opacity-50 disabled:cursor-not-allowed` |

Each row is a `div` with `flex items-center gap-2`:

```

```

- Qty input: `w-24 font-mono text-sm` (shadcn `Input`, `type="number"`, `step="any"`, `min="0.000001"`)
- UOM label: `text-xs text-muted-foreground min-w-[2.5rem] text-center` — reads from selected ingredient, not editable

#### Footer Controls

```

```

Footer uses `flex items-center gap-4 pt-4 border-t`.

#### Action Buttons (bottom of tab)

```
[POSButton variant="outline" touchSize="large"]  ← "Discard changes"
[POSButton variant="default" touchSize="large" type="submit"]  ← "Save recipe"  (or "Saving…" when pending)
```

Aligned `justify-end gap-2` — matches `ProductForm` footer.

#### States

| State | Visual |
|-------|--------|
| Empty (no recipe yet) | `EmptyState` from `@shared/ui` with `BookOpen` icon (lucide), title: "No recipe yet", description: "Add ingredients below to track depletion when this product is sold.", action: "+ Add ingredient" POSButton |
| Loading (fetching recipe) | `TableRowSkeleton` × 3 stacked in the rows area |
| Saving | Save button shows "Saving…" text, `disabled`, spinner using `LoadingSpinner` at `size="sm"` inside button |
| Save error | Inline `<p className="text-sm text-destructive">` above footer: "{error message}" |
| Save success | Toast: "Recipe saved." (sonner `toast.success`) |
| Row validation error | Qty input gets `border-destructive focus-visible:ring-destructive`; `<p className="text-xs text-destructive mt-1">` below field: "Quantity must be greater than 0" |

#### Depletion Preview Panel

```
<aside className="rounded-lg border bg-card p-4 space-y-3">
  <p className="text-sm font-semibold">
    Selling 1 {productName} will deplete:
  </p>
  {rows.map(row => (
    <div className="flex items-center justify-between gap-2 text-sm">
      <span>{ingredientName}</span>
      <span className="font-mono text-xs">{qty} {uom}</span>
      {stockIcon}  ← pos-accent checkmark | yellow AlertTriangle | destructive AlertTriangle
    </div>
  ))}
  {rows.length === 0 && (
    <p className="text-xs text-muted-foreground">Add at least one ingredient to see the preview.</p>
  )}
</aside>
```

Panel updates live on every keystroke/selection change. No debounce needed (pure computed derivation, no network call).

Low/negative stock warning row: appends `text-yellow-500` (low) or `text-destructive` (negative/out) class to the qty span, and renders `AlertTriangle` icon from lucide in the same color at `h-3 w-3`.

### Negative-Stock Toast + Override Trigger (`src/features/override-negative-stock/`)

**No new dialog component.** Reuses the existing `manager-pin-gate` feature modal.

#### Toast Message Format

When `add-item-to-tab` mutation returns `{ code: 'INVENTORY_NEGATIVE', ... }`:

```typescript
toast.error(`${ingredientName} is out of stock. Manager PIN required to override.`, {
  duration: 6000,
  action: {
    label: 'Allow override',
    onClick: () => openManagerPinGate({ context: 'override_negative_stock', payload: pendingOrderItem }),
  },
});
```

- Toast variant: `toast.error` (sonner destructive style — matches `--destructive` token)
- Duration: 6000ms (longer than default 4000ms — user needs time to read and act)
- Action button label: **"Allow override"** — two words, explicit intent
- If multiple ingredients are out of stock, list only the first in the toast; include count: "…and 2 more."

#### Override Flow

```
1. Toast appears: "Wings is out of stock. Manager PIN required to override." [Allow override]
2. User taps "Allow override" → existing ManagerPinGate modal opens
3. Manager enters PIN → mutation retries with allow_negative flag
4. On success → order item added; ledger row written; audit_log row written
5. Confirmation toast: "Override approved. Order item added." (sonner toast.success)
```

No new UI is built for this flow. The `override-negative-stock` feature slice contains only:
- The error handler logic (intercepts `INVENTORY_NEGATIVE` from `add-item-to-tab`)
- The retry mutation wrapper
- The toast trigger

#### Negative Stock Visual in Ledger

Ledger rows where `qty_delta < 0` and `current_stock < 0` after transaction:
- `text-destructive` for the stock level number
- `var(--pos-danger)` background tint: `bg-[color:var(--pos-danger)]/10` on the row

## Copywriting Contract

All copy strings are exact. No placeholder text permitted in production.

### CTAs

| Element | Copy | Variant |
|---------|------|---------|
| Add recipe row | "+ Add ingredient" | `POSButton variant="outline"` |
| Save recipe (idle) | "Save recipe" | `POSButton variant="default"` |
| Save recipe (saving) | "Saving…" | `POSButton variant="default" disabled` |
| Discard recipe edit | "Discard changes" | `POSButton variant="outline"` |
| Toast override action | "Allow override" | sonner action button |

### Empty States

| Location | Title | Description |
|----------|-------|-------------|
| Recipe tab — no recipe | "No recipe yet" | "Add ingredients below to track depletion when this product is sold." |
| IngredientAutocomplete — no results | — | "No ingredient found." (CommandEmpty — inline, no title) |
| Depletion preview — no rows | — | "Add at least one ingredient to see the preview." (inline paragraph, no title) |

### Error States

| Trigger | Message | Placement |
|---------|---------|-----------|
| Qty field empty or ≤ 0 | "Quantity must be greater than 0" | `text-xs text-destructive` below the qty input |
| No ingredient selected on row | "Select an ingredient" | `text-xs text-destructive` below the autocomplete trigger |
| Save failed (server error) | "Failed to save recipe. Please try again." | `text-sm text-destructive` above form footer |
| Ingredient is out of stock on sale | "{ingredientName} is out of stock. Manager PIN required to override." | sonner `toast.error` — 6s duration |
| Multiple ingredients out of stock | "{first ingredient} is out of stock (+{N} more). Manager PIN required to override." | sonner `toast.error` — 6s duration |

### Destructive Actions

| Action | Confirmation approach |
|--------|----------------------|
| Remove an ingredient row | Immediate (no dialog) — tap × removes the row inline; the recipe is not saved until "Save recipe" is tapped |
| Override negative stock | Manager PIN gate (existing modal) — not a `ConfirmDialog`, the PIN is the confirmation |

> **Design rationale:** Removing a row is not destructive until saved. The "Save recipe" button is the commit point, so per-row removal needs no confirm dialog.

## Accessibility Notes

- `IngredientAutocomplete` trigger: `aria-expanded`, `aria-haspopup="listbox"`, `role="combobox"` — handled by shadcn Command pattern
- Remove row button: `aria-label="Remove {ingredientName} from recipe"` — dynamic label for screen readers
- Depletion preview low/negative stock: `role="status"` on the aside, updated live — consumers get announced changes
- All touch targets ≥ 44px — enforced by `POSButton touchSize="large"` (48px) on "+ Add ingredient", "Discard changes", "Save recipe", and remove-row buttons; explicit `h-11` (44px) on `IngredientAutocomplete` trigger. `touchSize="default"` is NOT 48px — do not use it for interactive POS controls.
- Color is never the only differentiator — stock state icons (`Check`, `AlertTriangle`) accompany color indicators
