## Shared Patterns

### Exact-text/exact-name matching for Playwright locators
**Source:** `e2e/06-transfer.spec.ts` lines 29/36 (`{ exact: false }` on `getByText` scoped to a unique string) and the `exact: true` option on `getByRole`.
**Apply to:** Both E2E spec fixes in this phase (`06-transfer.spec.ts` T4, `09-rbac.spec.ts` T-RP-02). Rule: when a generic regex/substring locator can legitimately match 2+ elements (stacked toasts, overlapping action-name substrings), scope to the specific string rather than adding waits/sleeps.

### Static (non-interpolated) `aria-label` for singleton buttons
**Source:** `src/widgets/OrderPanel/ActiveTabSelector.tsx` lines 94, 106 (`"Transfer tab"`, `"New tab"`).
**Apply to:** `ActiveTabSelector.tsx`'s Split-bill button (D-04). Rule: a button that renders only once per view does not need dynamic/interpolated content in its `aria-label` for disambiguation — use a static string matching sibling buttons' convention, and reserve `data-testid` for any test that needs a stable hook.
