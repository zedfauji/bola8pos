Purpose: Single source of truth for routing (D-01) so `isFood`/`routing` can never disagree, and a safe migration path (D-03) that preserves existing categorization before dropping the old column.
Output: `CategoryRoutingSchema`/`CategoryRouting` in domain.ts, a `routing` field on `CategorySchema`, an idempotent migration file, and updated generated types.
