## Component Conformance Contract (COMPONENT-01/02/03)

### Decision rule â€” which primitive replaces which raw element

**Raw `<button>` â†’ one of two targets, chosen by role, never left raw:**

| Button role | Target | Rationale |
|---|---|---|
| Form CTA (Save/Cancel/Submit/Add-entity, sits in a `justify-end gap-2` footer) | `POSButton` (`@shared/ui/POSButton`), `touchSize="default"`, `variant` matching sibling CTAs already in the same file | Matches the exact pattern already used by every compliant Save/Cancel pair in this codebase (e.g. `ModifierGroupEditor.tsx:265-270`, `CategoryTreeEditor.tsx:122-127`) |
| Icon-only / text-link / toggle-chip / expand-collapse / card-tile utility control | `Button` (`@shared/ui/button`), variant chosen to preserve current visual: `ghost` for icon/hover-only controls, `link` for underlined text links, `outline` for pill/chip toggles, keep the full existing `className` string via `cn()` merge so pixel output is unchanged | Forcing `POSButton`'s 44px min-height + `active:scale-95` onto icon-only/chip/card controls would visually regress them (`POSButton` is sized for primary touch CTAs, not for a 14px underlined text link or a 160px nav tile) â€” `Button` is still a `shared/ui` primitive, satisfying "replace raw `<button>`" without altering appearance |

**Raw `<input>` (non-payment scope) â†’ per D-04/D-05/D-06/D-07, see table below.**

**Duplicate primitives (COMPONENT-03) â†’ remove the shadowing one-off**, see "New COMPONENT-03 finding" below.
