<objective>
Establish the TypeScript client contract layer for split payments — the Zod schemas, edge-function caller, and `processSplitPayment` wrapper that every downstream layer (PaymentForm UI, edge function response shape) builds against. This is the interface-first plan: it defines the request/response contract before the edge function or UI consume it.

Purpose: Give the PaymentForm UI (Plan 05) a typed, unit-tested `processSplitPayment(tabId, legs, expectedTotal, discountInfo?)` entry point, and lock the request/response envelope the new edge function (Plan 04) must satisfy. Honors D-11 by adding brand-new symbols only — the existing single-method processors and `callProcessPayment` are not touched.

Output: Extended `domain.ts`, `edge-function-contracts.ts`, `payment-processor.ts`, plus new split-payment unit tests in `payment-processor.test.ts`.
</objective>
