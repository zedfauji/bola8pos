## Dialog States

The dialog has four distinct states with unique layouts.

### State 1: Idle — Update Available

Shown at startup or after 4-hour poll when a new version is detected.

```
┌─────────────────────────────────────┐
│ Update Available            [badge: v2.1.0]    │
│ A new version is ready to install.             │
├─────────────────────────────────────┤
│ What's new                                      │
│ ┌─────────────────────────────────┐            │
│ │ [Changelog markdown — ScrollArea│            │
│ │  max-h-48, overflow-y-auto]     │            │
│ └─────────────────────────────────┘            │
├─────────────────────────────────────┤
│               [Remind Later] [Install Now]      │
└─────────────────────────────────────┘
```

- DialogTitle: "Update Available"
- DialogDescription: "Version {version} is ready to install."
- Badge: version string (e.g. "v2.1.0"), `variant="secondary"`, placed inline after the title
- Changelog: rendered inside `<ScrollArea className="max-h-48 rounded-md border p-3">`
- Changelog rendering: plain `<div className="prose prose-sm dark:prose-invert text-sm">` — render raw markdown as pre-formatted text (`whitespace-pre-wrap`) if `react-markdown` is not installed; do NOT add a new markdown library without explicit approval
- Footer: ghost "Remind Later" button on left, default "Install Now" button on right

### State 2: Downloading

Shown after user clicks "Install Now" while download + install is in progress.

```
┌─────────────────────────────────────┐
│ Downloading Update          [badge: v2.1.0]    │
│ Please wait — do not close the app.            │
├─────────────────────────────────────┤
│ [■■■■■■■░░░░░░░░░░░░░░] 42%         │
│ Downloading 42%                                 │
├─────────────────────────────────────┤
│                        [Cancel] (disabled)      │
└─────────────────────────────────────┘
```

- DialogTitle: "Downloading Update"
- DialogDescription: "Please wait — do not close the app."
- Progress bar: `<Progress value={percent} className="w-full" />` where `percent` is 0-100
- Progress label: `<p className="text-xs text-muted-foreground">Downloading {percent}%</p>` — updates every event tick from Tauri updater
- Both buttons disabled during download — "Install Now" shows `<Loader2 className="mr-2 h-4 w-4 animate-spin" />` spinner (matches ConfirmDialog loading pattern)
- Dialog is NOT dismissible during download (`onOpenChange` no-ops when `isDownloading` is true)

### State 3: Restart Ready

Shown after download and install completes, before restart.

```
┌─────────────────────────────────────┐
│ Ready to Restart            [badge: v2.1.0]    │
│ Update installed. Restart to apply changes.    │
├─────────────────────────────────────┤
│               [Later] [Restart Now]             │
└─────────────────────────────────────┘
```

- DialogTitle: "Ready to Restart"
- DialogDescription: "Update installed. Restart now to apply changes."
- No progress bar shown
- "Restart Now" button: `variant="default"` — calls Tauri relaunch
- "Later" button: `variant="ghost"` — dismisses dialog; app continues running with old binaries until next manual restart
- Dialog IS dismissible in this state (CONTEXT.md: no mid-service forced restart)

### State 4: Error

Shown if download fails (network drop, signature mismatch, etc.).

```
┌─────────────────────────────────────┐
│ Update Failed                                   │
│ Could not download the update. Try again later. │
├─────────────────────────────────────┤
│                               [Close]           │
└─────────────────────────────────────┘
```

- DialogTitle: "Update Failed"
- DialogDescription: "Could not download the update. Try again later."
- Single "Close" button (`variant="ghost"`) — dismisses dialog; silent retry on next scheduled check
- No version badge shown in error state

## Interaction Contract

| Trigger | Behavior |
|---------|----------|
| `onOpenChange(false)` in idle state | Calls `onRemindLater()` — hook stores "reminded" state, re-shows on next startup |
| `onOpenChange(false)` in downloading state | No-op — dialog is modal-locked during download |
| `onOpenChange(false)` in restart-ready state | Dismisses — app continues with installed update unapplied until manual restart |
| Keyboard Escape in idle state | Same as "Remind Later" — closes, sets reminded flag |
| Keyboard Escape in downloading state | Blocked — dialog non-dismissible |
| Keyboard Enter in idle state | Activates "Install Now" (default focused button) |
| "Install Now" click | Transitions to downloading state; calls `onInstall()` |
| "Remind Later" click | Calls `onRemindLater()`, closes dialog |
| "Restart Now" click | Calls `onRestart()` — triggers Tauri `relaunch()` |
| "Later" (restart state) click | Calls `onDismiss()`, closes dialog |
| "Close" (error state) click | Calls `onDismiss()`, closes dialog |

Focus management: Dialog uses Radix `Dialog` (not `AlertDialog`) — Radix traps focus automatically inside DialogContent. Initial focus lands on the primary action button ("Install Now" / "Restart Now") via `autoFocus` on that button.
