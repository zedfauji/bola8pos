## States Checklist (for executor)

- [ ] Loading: `LoadingSpinner` centered, matches `RecipeEditorTab.tsx:137-143` exactly
- [ ] Empty (0 rules): italic muted "No ingredient rules yet"
- [ ] Populated (N rules): row list, each row = ingredient autocomplete + signed numeric input + remove button
- [ ] Dirty/clean: Save button disabled until a change is made (mirrors `RecipeEditorTab.tsx`'s `disabled={!state.isDirty || isSaving}`)
- [ ] Saving: button shows `LoadingSpinner` + "Savingâ€¦" text, inputs disabled
- [ ] Save success: `toast.success('Ingredient rules saved')` (mirrors `toast.success('Modifier saved')` pattern) + dialog stays open or closes per planner's discretion (existing `RecipeEditorTab` stays on the tab; this is a dialog, so close-on-save is the closer analog to `ModifierDialog`'s existing behavior)
- [ ] Save failure: `toast.error(r.error.message)`, dialog stays open, no data loss
- [ ] Negative delta entry (e.g. `-1`): must round-trip correctly â€” this is the explicit regression case from Pitfall 3 in `17-RESEARCH.md`; executor must NOT use `MoneyInput` for this field
