## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → record_audit RPC | client supplies p_terminal_id + p_user_id (untrusted); actor authority still derives from auth.uid() unless p_user_id explicitly overrides |
| edge function (service-role) → audit_logs | service-role client bypasses RLS; actor_id must be supplied explicitly since auth.uid() is not meaningful there |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-05 | Repudiation | record_audit terminal/actor capture | mitigate | Add p_terminal_id + p_user_id so RPC-sourced rows carry terminal + actor; add actorId to edge helper so edge-sourced rows carry actor |
| T-14-06 | Spoofing | p_user_id override of actor | accept | v_actor_id := COALESCE(p_user_id, auth.uid()); callers that pass p_user_id are already trusted server paths (OfflineQueueProcessor). Standard RPC callers omit p_user_id and get auth.uid(). Low risk: audit_logs is append-only + SELECT is manager+ only |
| T-14-04 | Information Disclosure | record_audit failure | accept | EXCEPTION WHEN OTHERS → RAISE WARNING + RETURN NULL preserved; no internal error surfaced to caller |
| T-14-SC | Tampering | npm/pip/cargo installs | accept | Zero new packages this plan |
