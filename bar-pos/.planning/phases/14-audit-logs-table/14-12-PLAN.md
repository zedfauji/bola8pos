| login flow → auth.updateUser + clear_must_change_pin | the staff member changes their OWN credential; clear_must_change_pin only affects auth.uid() |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-03 | Elevation of Privilege | non-manager forcing a PIN change | mitigate | ProtectedAction UI gate (manage_staff) + force_pin_change RPC manager+ gate (authoritative) |
| T-14-15 | Denial of Service | staff locked out by a typo'd new PIN | mitigate | mandatory two-step enter+confirm; mismatch resets both fields; no single-entry path |
| T-14-12 | Elevation of Privilege | clearing another user's flag | mitigate | clear_must_change_pin operates on auth.uid() only (14-09); UI passes no target id |
| T-14-SC | Tampering | npm installs | accept | Zero new packages; AlertDialog/PINKeypad already installed |
