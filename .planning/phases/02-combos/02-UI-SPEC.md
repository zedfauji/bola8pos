## Spacing Scale

Declared values — Tailwind default 4px base unit, all multiples of 4:

| Token | Tailwind Class | Value | Usage |
|-------|---------------|-------|-------|
| xs | `gap-1` / `p-1` | 4px | Icon gaps, inline label padding |
| sm | `gap-2` / `p-2` | 8px | Compact element spacing, checkbox rows |
| md | `gap-4` / `p-4` | 16px | Default card padding, slot card internal padding |
| lg | `gap-6` / `p-6` | 24px | Sheet section padding, form section breaks |
| xl | `gap-8` / `p-8` | 32px | Layout gaps between major regions |
| 2xl | `gap-12` / `p-12` | 48px | Major section breaks (not used in this phase) |
| 3xl | `gap-16` / `p-16` | 64px | Page-level spacing (not used in this phase) |

Source: `bar-pos/src/shared/ui/Spacing.stories.tsx` (established project convention)

**Exceptions for this phase:**

| Exception | Value | Reason |
|-----------|-------|--------|
| POSButton touch target — default | 44px min-height | WCAG minimum; used for admin actions in ComboAvailabilityEditor and manage-combos list |
| POSButton touch target — large | 56px min-height | Comfortable bartender target; used for Confirm in ComboBuilderSheet |
| POSButton touch target — xl | 72px min-height | Primary action; NOT used in this phase (reserved for POS checkout) |
| QuantityControl buttons | 44px (h-11 w-11) | Existing pattern from `shared/ui/QuantityControl.tsx`; reused in ComboSlotCard qty selector |
| Day-of-week checkbox grid cell | min 44px height | Bartender-accessible touch target on the 7-day grid |

## Typography

All sizes use Geist Variable. Tailwind classes map to browser default 16px root.

| Role | Tailwind Class | Computed Size | Weight | Line Height | Usage |
|------|---------------|--------------|--------|-------------|-------|
| Display | `text-2xl font-bold` | 24px | 700 | 1.2 | Running total in ComboBuilderSheet header (`MoneyDisplay`) |
| Heading | `text-lg font-semibold` | 18px | 600 | 1.2 | Sheet title, section headers, combo name in admin list |
| Body | `text-base` (400) or `text-base font-medium` (500) | 16px | 400 / 500 | 1.5 | Slot labels, option names, availability window text |
| Label / Caption | `text-sm` / `text-xs` | 14px / 12px | 400 / 500 | 1.4 | Min/max hints below slot card, badge text, muted metadata |

**Weights in use: regular (400) and semibold (600).** Font-medium (500) is used only for `text-base font-medium` item totals (existing ModifierSheet pattern) and is not a third weight — Geist Variable renders it from the same file.

Source: `bar-pos/src/shared/ui/Typography.stories.tsx`

## Color

All color values reference CSS custom properties from `bar-pos/src/app/globals.css`. Dark mode is default (system preference via `prefers-color-scheme: dark`).

| Role | CSS Variable | Dark Mode Value | Usage |
|------|-------------|-----------------|-------|
| Dominant (60%) | `--background` | `oklch(0.145 0 0)` — near-black | Page background, Sheet backdrop, admin page |
| Secondary (30%) | `--card` / `--muted` | `oklch(0.205 0 0)` / `oklch(0.269 0 0)` | ComboSlotCard surface, unavailable product card, admin list rows |
| Accent (10%) | `--primary` | `oklch(0.922 0 0)` — near-white | Reserved list below |
| Destructive | `--destructive` | `oklch(0.704 0.191 22.216)` — red | Delete combo, nested-combo error toast |
| POS accent (availability) | `--pos-accent` | `oklch(0.78 0.2 145)` — green | Active availability day chips, ComboBadge when combo is available |
| POS danger (unavailable) | `--pos-danger` | `oklch(0.7 0.22 25)` — red-orange | ComboUnavailableBadge, greyed-out unavailable product card overlay hint |

**Accent (`--primary`) reserved for:**
- ComboBuilderSheet: "Add to Order" confirm button (primary variant)
- ComboAvailabilityEditor: "Save" button (primary variant)
- manage-combos admin: "Add combo" and "Save" buttons (primary variant)
- Active/selected state of day-of-week checkboxes (shadcn Checkbox — uses `--primary` ring)
- ComboSlotCard selected child: option row background highlight

Accent is NOT used for: text labels, borders, metadata, muted hints, or the CollapsibleParent chevron in tab view.

Source: `bar-pos/src/app/globals.css` + `bar-pos/tailwind.config.ts` — all existing tokens, no new tokens needed.

## Component Inventory

### New shared/ui components (require Storybook story)

#### `ComboSlotCard` (`src/shared/ui/ComboSlotCard/ComboSlotCard.tsx`)

- Surface: `bg-card rounded-lg border p-4`
- Header row: slot label (`text-base font-medium`) + min/max hint (`text-xs text-muted-foreground`)
- Option list: scrollable `max-h-48 overflow-y-auto divide-y` — each row is `flex items-center gap-3 px-3 py-2`
- Each option row: shadcn `Checkbox` + `Label` + price delta (right-aligned, `text-sm font-medium`)
- Selected state: option row gets `bg-accent/50` background
- Qty selector: when `min_qty === max_qty`, show read-only qty display. When range, show `QuantityControl` (reuse existing `shared/ui/QuantityControl`)
- Footer: running sub-total for this slot — `text-sm text-muted-foreground` + `MoneyDisplay`
- Validation state: when required slot has 0 selections → red border `border-destructive`
- Reuses: `Checkbox` (shadcn), `Label` (shadcn), `QuantityControl` (shared/ui), `MoneyDisplay` (shared/ui)
- Props: `slot: ComboSlot`, `options: ComboSlotOption[]`, `value: SlotSelection`, `onChange: (v: SlotSelection) => void`, `disabled?: boolean`

#### `ComboBadge` (`src/shared/ui/ComboBadge.tsx`)

- `Badge` variant `secondary` + `bg-pos-accent/20 text-pos-accent border-pos-accent/30` override
- Text: "Combo" (fixed string, non-translatable in v1)
- Size: `text-xs px-1.5 py-0.5`
- Used on product cards in ProductGrid when `product.is_combo === true`
- No Storybook story required (it's a trivial one-liner wrapping Badge) — EXCEPTION: the spec calls for Storybook on new shared/ui components; add a minimal story.

#### `ComboUnavailableBadge` (`src/shared/ui/ComboUnavailableBadge.tsx`)

- `Badge` variant `secondary` + `bg-muted text-muted-foreground` (greyed)
- Contains: lock icon (`Lock` from lucide, 12px) + availability text
- Availability text format: "Available Mon–Fri" or "Available Mon 18:00–22:00"
- Screen-reader: `aria-label="Combo unavailable. {full reason}"` on the outer element
- Color contrast requirement: muted-foreground on muted background must pass ≥ 4.5:1 in dark mode. Existing `--muted-foreground: oklch(0.708 0 0)` on `--muted: oklch(0.269 0 0)` passes in dark mode.
- Props: `availabilityHint: string` (pre-formatted by caller)

### New feature UI components (no Storybook required)

#### `ComboBuilderSheet` (`src/features/add-combo-to-tab/ui/ComboBuilderSheet.tsx`)

- Pattern: extends `ModifierSheet` — uses `Sheet` with `side="bottom"` and `h-[80vh]`
- Header: `SheetTitle` = combo product name; `SheetDescription` = "Select options for each slot"
- Running total bar: same as ModifierSheet — `flex items-center justify-between border-t pt-4 mt-2`; running total uses `MoneyDisplay size="lg"`
- Body: `max-h-[calc(80vh-240px)] overflow-y-auto space-y-4 py-6` — one `ComboSlotCard` per slot, ordered by `sort_order`
- Pool-time slots: rendered as a non-interactive info card — "1 hour pool time included" — no picker needed
- Footer: two buttons full-width via `SheetFooter gap-2`
  - Cancel: `Button variant="outline" flex-1`
  - "Add to Order": `POSButton touchSize="large" flex-1` — **disabled** until all required slots filled
- Disabled confirm state: button has `disabled` attr; no tooltip needed (slot cards already show red border on unfilled required slots)
- Availability override path: if manager PIN override was granted before sheet opened, show yellow `Alert` banner at top of sheet: "Manager override active — availability check bypassed"

#### `ComboAvailabilityEditor` (`src/features/manage-combos/ui/ComboAvailabilityEditor.tsx`)

- Container: `space-y-6`
- Day-of-week grid: 7 cells in a `flex flex-wrap gap-2`; each cell is a toggle button (shadcn `Button variant="outline"` with `data-selected` → switches to `variant="default"`)
  - Day labels: 3-letter English abbreviations: Mon Tue Wed Thu Fri Sat Sun
  - Touch target: min 44px (use `POSButton touchSize="default"`)
  - Selected day: primary fill (`variant="default"`)
  - Unselected day: outline (`variant="outline"`)
- Time window: conditional on at least one day selected
  - Two `<input type="time">` fields for `start_time` and `end_time` — styled same as `DateRangePicker` inputs: `rounded-md border border-input bg-background px-2 py-1 text-sm focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-ring`
  - Label: `text-sm text-muted-foreground` — "Available from" / "to"
  - Leave empty = all day (both null in DB)
- Date range: reuse `DateRangePicker` from `shared/ui/DateRangePicker`; leave empty = no date restriction (both null)
- Multiple availability windows: "Add window" button adds a new row; each row has a trash icon to remove
  - Each row is self-contained: days + time window + date range
  - "Add window" = `POSButton touchSize="default" variant="outline"` with `Plus` icon
- Note below editor: `text-xs text-muted-foreground` — "No windows = always available"

### Modified existing components

#### `ProductGrid` (`src/widgets/ProductGrid/`)

- Add "Combos" category filter chip — rendered same as existing category filter chips
- Product card when `is_combo === true` and combo is available: render `ComboBadge` in top-right corner of card
- Product card when `is_combo === true` and combo is unavailable: 
  - Card opacity: `opacity-60`
  - `ComboUnavailableBadge` rendered below product name
  - Card still tappable (opens sheet with unavailability message) — manager can then invoke override
- On tap of unavailable combo card: show `ComboUnavailableDialog` (shadcn `Dialog`) with reason + "Request manager override" button → triggers `manager-pin-gate` flow

#### KDS (existing entity)

- Items with `parent_order_item_id` are grouped under parent
- Parent line: product name + `ComboBadge` + total qty (`text-base font-semibold`)
- Expand/collapse: shadcn `Collapsible` with `ChevronDown` icon (rotates 180° when open)
- Child lines: indented `pl-6`, `text-sm`, price shown as 0 or suppressed

#### Tab order view (existing widget)

- Parent combo order_item: renders as collapsible row — `ChevronDown` icon right side
- Collapsed: "Cubeta Regular x1 — $450.00" in `text-base font-medium`
- Expanded: children listed with `pl-6 text-sm text-muted-foreground`, each showing child name + qty
- Children are read-only (cannot be individually voided in S2 — that's S4)

## Interaction Contracts

### ComboBuilderSheet

| Interaction | Behaviour |
|-------------|-----------|
| Open trigger | Tap product card with `is_combo=true` and available — replaces straight `add-item-to-tab` call |
| Close | Swipe down OR tap Cancel — resets all slot selections |
| Slot selection | Tap option row toggles checkbox; `QuantityControl` only shown when `min_qty !== max_qty` |
| Required slot unfilled | Red border on `ComboSlotCard`, confirm button disabled |
| All slots filled | Confirm button enabled; running total updates in real-time |
| Confirm | Calls `add_combo_to_tab` RPC; on success: sheet closes, toast "Added {combo name}"; on error: toast with specific message |
| Pool-time slot | Non-interactive info card — no selection needed |

### Unavailable combo tap

| State | Behaviour |
|-------|-----------|
| Tap greyed card | Opens `ComboUnavailableDialog` |
| "Request override" | Triggers existing `manager-pin-gate` pattern |
| Manager PIN accepted | Sheet opens with yellow override banner, `override_availability=true` in RPC payload |
| Manager PIN rejected | Dialog closes, card remains greyed |

### ComboAvailabilityEditor (admin)

| Interaction | Behaviour |
|-------------|-----------|
| Day toggle | Click day button toggles selected state; ISO 1–7 values stored |
| No days selected | Time/date inputs hidden |
| Time inputs | HTML `<input type="time">` — browser native picker (consistent with existing `DateRangePicker` approach) |
| Both time fields empty | Saved as null — interpreted as "all day" |
| End time before start time | Inline validation: `text-sm text-destructive` "End time must be after start time" |
| Date range optional | Both fields empty = no date restriction (null in DB) |
| Add window | Appends new row with all-days pre-checked, no time restriction |
| Remove window | `ConfirmDialog` only if window was previously saved; immediate removal if unsaved |
| Save | Upserts `combo_availability` rows for this combo; toast "Availability saved" on success |

### KDS collapsible bundle

| Interaction | Behaviour |
|-------------|-----------|
| Default state | Collapsed — shows parent line only |
| Tap chevron | Expands to show child lines; chevron rotates 180° |
| Tap again | Collapses; state is local UI (not persisted) |

## Accessibility Contract

Source: `04-navigation-ui-flows.md § Accessibility & UX guardrails`

| Requirement | Implementation |
|------------|----------------|
| All interactive elements keyboard-reachable | shadcn Sheet and Dialog have built-in focus traps; day buttons are native `<button>` elements; slot option rows use shadcn `Checkbox` + `Label` |
| Focus trap in Sheet | shadcn Sheet default — do NOT override `onOpenAutoFocus` or `onCloseAutoFocus` |
| Focus trap in Dialog | shadcn Dialog default — same |
| Manager PIN modal | Existing `manager-pin-gate` pattern — already compliant; clear input on dismiss |
| `ComboUnavailableBadge` screen-reader text | `aria-label="Combo unavailable. {availabilityHint}"` on root element |
| Color contrast on badges | `--muted-foreground` on `--muted` in dark mode: ≥ 4.5:1 ratio (verified by existing token values) |
| No single-tap-without-confirm on destructive | Delete combo and Remove slot both go through `ConfirmDialog` |
| Confirm button disabled state | Native HTML `disabled` attribute — screen reader announces as "dimmed" / "greyed out"; no ARIA needed beyond native |
| QuantityControl | Existing `aria-live="polite"` on qty display; `aria-label` on +/– buttons — reused as-is |
