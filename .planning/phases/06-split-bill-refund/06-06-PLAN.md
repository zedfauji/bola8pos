**SubTabColumn.tsx** — implement per UI-SPEC exactly:

Props interface:
```typescript
export interface SubTabColumnProps {
  label: string;           // "Alice", "Sub-tab 1", etc.
  items: OrderItem[];      // items currently assigned to this column
  total: number;           // running sum in CENTS (integer) — display via MoneyDisplay
  isSelected: boolean;     // receives the next item tap
  onSelect: () => void;    // parent sets this as the active drop target
  onRemoveItem: (itemId: string) => void;
}
```

Structure per UI-SPEC:
```tsx
<div
  role="option"
  aria-selected={isSelected}
  className={cn(
    'flex flex-col min-w-[140px] bg-card rounded-lg border cursor-pointer transition-colors',
    isSelected ? 'border-primary ring-2 ring-primary/30' : 'border-border'
  )}
  onClick={onSelect}
>
  {/* ColumnHeader */}
  <div className="flex items-center justify-between px-3 py-2 border-b">
    <span className="text-sm font-semibold truncate max-w-[100px]">{label}</span>
    <MoneyDisplay amount={total / 100} size="sm" />
  </div>

  {/* ItemList */}
  <div className="flex-1 overflow-y-auto divide-y min-h-[120px]">
    {items.map(item => (
      <div
        key={item.id}
        className="flex items-center justify-between px-3 py-2 gap-2 hover:bg-accent/30"
        onClick={e => e.stopPropagation()}  // prevent column selection when clicking item
      >
        <span className="text-sm truncate flex-1">{item.productName}</span>
        <span className="text-sm text-muted-foreground font-mono">×{item.qty}</span>
        <MoneyDisplay amount={item.unitPrice} size="sm" />
        <button
          aria-label={`Remove ${item.productName} from ${label}`}
          className="text-muted-foreground hover:text-destructive transition-colors"
          onClick={e => { e.stopPropagation(); onRemoveItem(item.id); }}
        >
          <X className="h-3.5 w-3.5" />
        </button>
      </div>
    ))}
  </div>

  {/* DropZone — hidden when column has items */}
  {items.length === 0 && (
    <div className="px-3 py-2 text-sm text-muted-foreground text-center border-t">
      Tap an item to assign here
    </div>
  )}
</div>
```

Note: `item.productName` and `item.unitPrice` — check the actual field names on `OrderItem` from domain.ts (may be `product_name`, `unit_price`, or camelCase variants). Use the correct field names from the Zod schema.

**PersonCard.tsx** — extend SubTabColumn by replacing the static label with an editable Input:

```typescript
import type { SubTabColumnProps } from '@shared/ui/SubTabColumn';
import { SubTabColumn } from '@shared/ui/SubTabColumn';
import { Input } from '@shared/ui/input';
import type { OrderItem } from '@shared/lib/domain';

export interface PersonCardProps extends Omit<SubTabColumnProps, 'label'> {
  name: string;
  onNameChange: (name: string) => void;
}
```

PersonCard cannot simply wrap SubTabColumn as-is because the label area needs to be different. Two implementation options:

**Option A (preferred if SubTabColumn supports labelSlot):** If SubTabColumn accepts a `labelSlot?: React.ReactNode` prop, PersonCard passes the Input as the label slot.

**Option B (copy-paste + replace):** PersonCard copies SubTabColumn's structure but replaces `<span>{label}</span>` with:

```tsx
<Input
  value={name}
  onChange={e => onNameChange(e.target.value)}
  placeholder="Person name"
  className="text-sm font-semibold h-7 border-0 px-1 bg-transparent focus-visible:ring-1"
  maxLength={30}
  onClick={e => e.stopPropagation()}  // don't trigger column selection when editing name
/>
```

Choose Option A if you can refactor SubTabColumn to accept `labelSlot`. Otherwise use Option B. The name Input should auto-focus when the PersonCard is newly added (use `autoFocus` prop when the component mounts).
