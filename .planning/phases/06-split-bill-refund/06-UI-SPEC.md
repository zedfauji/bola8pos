## Visual Hierarchy

**SplitTabSheet:** Mode tab bar (Evenly/Item/By Person/By Amount) is the primary nav element at the top of the Sheet. The "Confirm Split" `POSButton` at the Sheet footer is the single primary focal point — only enabled when valid. Disabled state uses native HTML `disabled` + `opacity-50`.

**RefundSheet:** The item list (with qty spinners) occupies the body. The running refund total (`MoneyDisplay size="lg"`) appears in a sticky bar above the Sheet footer, visible at all scroll positions. The "Request approval" button is the sole accent CTA.

**PaymentsPage:** The "Refund" button in paid rows uses `variant="destructive"` sizing `size="sm"` to visually distinguish it from the neutral row text. It appears only when the payment is refundable (rules below).

#### `SubTabColumn` (`src/shared/ui/SubTabColumn/SubTabColumn.tsx`)

Represents one sub-check column in the Item and By Person split modes.

**Structure:**
```
SubTabColumn (flex flex-col min-w-[140px] bg-card rounded-lg border)
  ColumnHeader (flex items-center justify-between px-3 py-2 border-b)
    label (text-sm font-semibold truncate max-w-[100px])
    total (MoneyDisplay size="sm")
  ItemList (flex-1 overflow-y-auto divide-y min-h-[120px])
    [ItemRow × N] — see below
  DropZone (px-3 py-2 text-sm text-muted-foreground text-center border-t)
    "Tap an item to assign here"  — hidden when column has ≥ 1 item
```

**ItemRow within SubTabColumn:**
```
div.flex.items-center.justify-between.px-3.py-2.gap-2 + hover:bg-accent/30 + cursor-pointer
  span.text-sm.truncate  — item name
  span.text-sm.text-muted-foreground.font-mono  — "×{qty}"
  MoneyDisplay size="sm"
  button[aria-label="Remove {itemName} from {columnLabel}"] — X icon (lucide X, 14px)
    hover:text-destructive; tap removes item back to Unassigned column
```

**Tapping an assigned item in a column** removes it back to Unassigned. There is no drag-and-drop — assignment is done by: (1) tap item in Unassigned → item is "selected" (ring-2 ring-primary on row); (2) tap a SubTabColumn → item moves to that column. Selected item deselects if user taps Unassigned again or presses Escape.

**Props:**
```typescript
interface SubTabColumnProps {
  label: string;           // "Alice", "Sub-tab 1", etc.
  items: OrderItem[];      // items currently assigned to this column
  total: number;           // running sum of items in column
  isSelected: boolean;     // receives the next item tap
  onSelect: () => void;    // parent sets this as the active target column
  onRemoveItem: (itemId: string) => void;
}
```

**States:**
- Default: `border-border`
- Selected (active drop target): `border-primary ring-2 ring-primary/30` — signals that tapping an item will assign it here
- Empty (no items): shows drop zone hint text
- Full (at least 1 item): hides drop zone hint; shows items list

**Storybook story variants:** `Empty`, `Selected` (ring highlight), `WithItems` (3 items), `SingleItem`

##### Mode: Evenly

```
TabsContent[value="evenly"] (px-6 py-4 flex flex-col items-center gap-6)
  <p className="text-sm text-muted-foreground">How many people are splitting this tab?</p>

  [N keypad — 1–9 digit grid, 3 columns]
  <div className="grid grid-cols-3 gap-3 w-full max-w-[280px]">
    {[1,2,3,4,5,6,7,8,9].map(n => (
      <POSButton
        key={n}
        touchSize="xl"
        variant={selected === n ? "default" : "outline"}
        onClick={() => setN(n)}
        aria-label={`Split ${n} ways`}
        aria-pressed={selected === n}
      >
        {n}
      </POSButton>
    ))}
  </div>

  [Preview section — visible only when N ≥ 2]
  <div className="rounded-lg bg-card border p-4 w-full space-y-2">
    <p className="text-sm font-semibold">Preview</p>
    <div className="flex items-center justify-between">
      <span className="text-sm text-muted-foreground">{N−1} × payments</span>
      <MoneyDisplay amount={baseAmount} size="sm" />
    </div>
    <div className="flex items-center justify-between">
      <span className="text-sm text-muted-foreground">1 × payment (absorbs rounding)</span>
      <MoneyDisplay amount={lastAmount} size="sm" />
    </div>
    <div className="flex items-center justify-between border-t pt-2">
      <span className="text-sm font-semibold">Total</span>
      <MoneyDisplay amount={tab.total} size="md" />
    </div>
  </div>
```

**Rounding display contract (P8/P9 precision):**
- `baseAmount = Math.floor(tab.total * 100 / N) / 100` (floor to cent)
- `lastAmount = tab.total - baseAmount * (N - 1)` (absorbs any remaining cent)
- Both displayed via `MoneyDisplay` — renders to 2 decimal places via `formatMoney()`
- When `N = 1`: shows no preview (single payment = full tab; disable Confirm)
- `isValid` for Evenly mode: `N >= 2`

**Confirm behavior:** Does NOT create sub-tabs. Calls `split_tab_evenly` RPC which returns `{ per_payment_amount, cents_remainder }`. Then caller inserts N payment rows. Sheet closes on success.

##### Mode: Item

```
TabsContent[value="item"] (flex flex-col h-full)
  [Column area — horizontal scroll]
  <div className="flex gap-3 overflow-x-auto px-6 pb-3 flex-1">
    [UnassignedColumn — always leftmost, non-removable]
    <SubTabColumn label="Unassigned" items={unassigned} ... />

    {columns.map(col => <SubTabColumn key={col.id} ... />)}

    [Add column button]
    <button
      className="min-w-[140px] flex flex-col items-center justify-center gap-2 rounded-lg border border-dashed text-muted-foreground hover:text-foreground hover:border-foreground transition-colors"
      onClick={addColumn}
      aria-label="Add sub-check column"
    >
      <Plus className="h-5 w-5" />
      <span className="text-sm">Add check</span>
    </button>
  </div>

  [Unassigned warning — shown when confirmAttempted && unassigned.length > 0]
  <p className="px-6 text-sm text-pos-danger">
    {unassigned.length} item(s) not assigned. Assign all items before confirming.
  </p>
```

**Item selection and assignment UX (tap-to-assign):**
1. User taps an item row in any column — item gains `ring-2 ring-primary` selection state; a "selected item" indicator appears at the top of the Sheet content area showing the item name.
2. User taps a SubTabColumn (anywhere on the column surface, not just a drop zone) — if an item is selected AND the column is not Unassigned AND the item is not already in that column, item moves there.
3. Tapping the Unassigned column while an item is selected moves the item back to Unassigned.
4. Tapping the same item again deselects it (ring removed).
5. Pressing Escape deselects the current item.
6. The X button on an assigned item row removes it directly to Unassigned without requiring a second tap.

**Selected item indicator bar (shown while an item is selected):**
```
<div className="mx-6 mb-3 flex items-center gap-2 rounded-md bg-primary/10 border border-primary/30 px-3 py-2">
  <span className="text-sm text-primary">"{selectedItem.name}" selected</span>
  <span className="text-sm text-muted-foreground ml-auto">Tap a column to assign</span>
</div>
```

**Minimum columns:** 2 (at least 1 non-Unassigned column required). "Add check" button always visible.

`isValid` for Item mode: `unassigned.length === 0 && columns.length >= 1`

##### Mode: By Person

Visually identical to Item mode, but uses `PersonCard` instead of `SubTabColumn` for the non-Unassigned columns. Same tap-to-assign UX.

**Difference from Item mode:** Items that remain in Unassigned at confirm time are NOT a validation error — they are split evenly across all N persons by the RPC. This is the only mode where Unassigned items are allowed at confirm time.

The Unassigned column header shows: "Unassigned (split evenly)" — `text-sm text-muted-foreground italic`.

`isValid` for By Person mode: `columns.length >= 2` (at least 2 persons; Unassigned items are allowed)

##### Mode: By Amount

```
TabsContent[value="amount"] (px-6 py-4 flex flex-col gap-4)
  [Warning banner]
  <Alert variant="default" className="border-yellow-500/30 bg-yellow-500/5">
    <AlertDescription className="text-sm text-muted-foreground">
      Amount split does not preserve item-level reporting. Use By Item when possible.
    </AlertDescription>
  </Alert>

  [Inputs list]
  <div className="space-y-3">
    {amountRows.map((row, i) => (
      <div key={row.id} className="flex items-center gap-3">
        <Input
          placeholder={`Check ${i + 1} label`}
          value={row.label}
          onChange={e => updateLabel(row.id, e.target.value)}
          className="flex-1"
          maxLength={30}
        />
        <MoneyInput
          value={row.amount}
          onChange={v => updateAmount(row.id, v)}
          className="w-28"
          placeholder="0.00"
        />
        {amountRows.length > 2 && (
          <button onClick={() => removeRow(row.id)} aria-label={`Remove check ${i + 1}`}>
            <X className="h-4 w-4 text-muted-foreground hover:text-destructive" />
          </button>
        )}
      </div>
    ))}
    <Button variant="outline" size="sm" onClick={addRow} className="w-full">
      <Plus className="h-4 w-4 mr-2" /> Add check
    </Button>
  </div>

  [Running total and remaining]
  <div className="rounded-lg bg-card border p-4 space-y-2">
    <div className="flex items-center justify-between">
      <span className="text-sm text-muted-foreground">Entered total</span>
      <MoneyDisplay amount={enteredTotal} size="sm" className={remainingExact ? "" : "text-pos-danger"} />
    </div>
    <div className="flex items-center justify-between">
      <span className="text-sm text-muted-foreground">Remaining</span>
      <MoneyDisplay amount={remaining} size="sm" className={remaining === 0 ? "text-pos-accent" : "text-pos-danger"} />
    </div>
    <div className="flex items-center justify-between border-t pt-2">
      <span className="text-sm font-semibold">Tab total</span>
      <MoneyDisplay amount={tab.total} size="md" />
    </div>
  </div>
```

**Live remaining logic:**
- `enteredTotal = sum(amountRows.map(r => r.amount))` — always reflects current inputs
- `remaining = tab.total - enteredTotal`
- `remainingExact = Math.abs(remaining) <= 0.01` (allow ±1 cent tolerance per RPC contract)
- "Remaining" value color: `text-pos-accent` when `remaining === 0`; `text-pos-danger` when `remaining > 0` or `remaining < 0`

`isValid` for By Amount mode: `remainingExact && amountRows.every(r => r.amount > 0) && amountRows.length >= 2`

## Interaction Contracts

### SplitTabSheet — state machine

```
CLOSED
  → [Split bill button tap on open tab] → OPEN (default mode: Evenly)

OPEN/Evenly
  Substates: N_UNSET (N=0), N_SET (N >= 2), INVALID (N=1)
  → tap digit 2–9 → N_SET; updates preview
  → tap digit 1 → INVALID (Confirm disabled)
  → Confirm (N >= 2) → CALLING_RPC → [success] → CLOSED + success toast
                                   → [error] → OPEN/Evenly + error toast

OPEN/Item
  Substates: IDLE, ITEM_SELECTED
  IDLE
    → tap item (in any column) → ITEM_SELECTED (item rings with primary border)
  ITEM_SELECTED
    → tap SubTabColumn → item moves to column; → IDLE
    → tap Unassigned column → item moves back → IDLE
    → tap same item → IDLE (deselect)
    → Escape key → IDLE
  IDLE (all items assigned)
    → Confirm → CALLING_RPC → [success] → CLOSED + success toast
                             → [error] → OPEN/Item + error toast
  IDLE (any items unassigned)
    → Confirm attempt → shows unassigned warning (stays OPEN/Item)

OPEN/ByPerson
  [Identical to Item with additions:]
  → Add person button → adds PersonCard; auto-focus name input
  → Name input blur/submit → updates label (optimistic, pre-split)
  → Confirm (N >= 2 persons) → CALLING_RPC → [success] → CLOSED + success toast
    [Unassigned items at confirm are split evenly by RPC — NOT a block]

OPEN/ByAmount
  Substates: EDITING, VALID (remaining === 0 ± 1 cent), INVALID
  → update any amount input → recalculate remaining; color flip
  → remaining within ±1 cent → VALID; Confirm enabled
  → Confirm → CALLING_RPC → [success] → CLOSED + success toast
                           → [error] → OPEN/ByAmount + error toast
```

### RefundSheet — state machine

```
CLOSED
  → [Refund button tap on refundable payment row] → OPEN (all items deselected)

OPEN
  Substates: SELECTING (no items selected), CONFIGURING (items selected), SUBMITTING

  SELECTING
    → tap Checkbox on item → CONFIGURING
  CONFIGURING
    → toggle more items; adjust qtys; set reason; toggle restock
    → deselect all → SELECTING
    → "Request approval" (valid: items selected + reason set) → PIN_MODAL

  PIN_MODAL (renders ManagerPinDialog)
    → PIN accepted → SUBMITTING
    → PIN dismissed → CONFIGURING (sheet stays open; PIN modal closes)

  SUBMITTING
    → Sheet buttons disabled; spinner on "Request approval"
    → [RPC success] → CLOSED + success toast + optional receipt reprint prompt
    → [RPC error] → CONFIGURING + error toast

  RECEIPT_REPRINT_PROMPT (post-success Dialog)
    → "Print receipt" → triggers Tauri IPC print
        → [print success] → dismissed; no further action
        → [print failure] → error toast "Receipt print failed — check printer connection."
    → "Skip" → dismissed; no action
```

### Qty spinner in RefundSheet

- Uses existing `QuantityControl` component with `min={1}` and `max={item.original_qty - item.already_refunded_qty}`
- `already_refunded_qty` is computed from existing `refund_items` for this payment: `sum(refund_items.qty where order_item_id = item.id)`
- If `max === 0`, the checkbox for that item is disabled with `title="Fully refunded"` and the item row gets `opacity-50`

### Sub-tab visibility in tab detail

When `tab.status === 'split'`: hide the "Split bill" button entirely; show sub-checks panel (see component spec above). Sub-tabs with `status === 'paid'` show `text-pos-accent` label + paid StatusBadge.

## Accessibility Contract

Inherited from locked decisions in `04-navigation-ui-flows.md`:

| Requirement | Implementation |
|------------|----------------|
| All interactive elements keyboard-reachable | shadcn Sheet, Dialog, Tabs, Checkbox use native keyboard patterns; digit buttons in Evenly mode are `<button>` elements |
| Focus trap in Sheet | shadcn Sheet default — do NOT override `onOpenAutoFocus` or `onCloseAutoFocus` |
| Focus trap in ManagerPinDialog | Existing `manager-pin-gate` pattern — already compliant; clear input on dismiss |
| No single-tap-without-confirm for destructive | Refund requires manager PIN. Cancel with assignments requires ConfirmDialog. |
| Evenly mode digit buttons | `aria-pressed={selected === n}` on each button; `aria-label="Split {n} ways"` |
| SubTabColumn selection state | `aria-selected` on selected column (role="option" on columns, role="listbox" on column container) |
| RefundItemRow checkbox | `aria-label="Include {product_name} in refund"` |
| Restock checkbox | `aria-label="Restock {product_name}"` |
| Qty spinner | Reuses existing `QuantityControl` aria-live + aria-label pattern |
| SplitTabSheet mode tabs | shadcn `Tabs` — native `role="tablist"` / `role="tab"` / `role="tabpanel"`; no override needed |
| Confirm button disabled state | Native HTML `disabled` attribute — screen reader announces as unavailable; no additional ARIA |
| MoneyDisplay | `aria-label="Negative {amount} dollars"` already implemented in MoneyDisplay component |
| Color contrast | All text/surface combinations use existing CSS variable pairs verified ≥ 4.5:1 in dark mode |
| "Remaining" live update | Wrap "Remaining: $X.XX" container with `aria-live="polite"` — announced on change |
| Sub-tab status in tab detail | `StatusBadge` with `aria-label="{label} — {status}"` on each sub-tab row |
