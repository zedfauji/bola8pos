## Spacing Scale

All values are multiples of 4. Match existing SettingsPage and manage-combos patterns.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`gap-1`, `p-1`) | Icon gaps, badge padding, swatch gutters |
| sm | 8px (`gap-2`, `p-2`) | Inline row action gaps, input group spacing |
| md | 16px (`gap-4`, `p-4`) | Default form field spacing, tab content padding |
| lg | 24px (`gap-6`, `p-6`) | Section breaks inside Dialog/Sheet |
| xl | 32px (`gap-8`, `p-8`) | Page-level padding (SettingsPage uses `p-6 md:p-8`) |
| 2xl | 48px | Major section separations (ledger header from table) |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Row padding inside DataTable body rows: `py-2.5 px-3` (10px / 12px) — matches ManageCombosTab list pattern
- Empty state vertical padding: `py-10` (40px) — matches existing EmptyState component (`py-12`)
- Dialog max-width: `max-w-lg sm:max-w-lg` — matches ComboBuilderForm Dialog

## Typography

Match SettingsPage and manage-categories patterns. Font is Geist Variable throughout.

| Role | Size | Weight | Line Height | Tailwind |
|------|------|--------|-------------|---------|
| Body | 14px | 400 (regular) | 1.5 | `text-sm` |
| Label | 14px | 500 (medium) | 1.4 | `text-sm font-medium` |
| Heading (tab section) | 16px | 600 (semibold) | 1.4 | `text-base font-semibold` |
| Page heading | 30px | 700 (bold) | 1.2 | `text-3xl font-bold` (inherited from SettingsPage `<h1>`) |

Exactly 3 sizes in this phase: 14px (body/label), 16px (section heading), 30px (page h1 — inherited, not re-declared).
Exactly 2 weights used in new components: regular 400 + semibold 600 (medium 500 for form labels only).

Muted text: `text-muted-foreground text-sm` — used for descriptions, timestamps, helper copy.
Mono for numeric values (stock qty, cost, factor): `font-mono text-sm` — ensures alignment in DataTable cells.

## Color

All values reference CSS variables from `globals.css`. Dark mode is default (system).

| Role | CSS Variable | Dark mode value | Usage |
|------|-------------|-----------------|-------|
| Dominant surface (60%) | `--background` | `oklch(0.145 0 0)` — near-black | Page background, tab content areas |
| Secondary surface (30%) | `--card` / `--muted` | `oklch(0.205 0 0)` / `oklch(0.269 0 0)` | DataTable rows, Dialog background, Sheet panel |
| Accent (10%) | `--primary` | `oklch(0.922 0 0)` — near-white | Primary CTAs only (see reserved list below) |
| Destructive | `--destructive` | `oklch(0.704 0.191 22.216)` — red | Delete ingredient confirm button only |
| Positive delta | `--pos-accent` | `oklch(0.78 0.2 145)` — green | Stock movement delta cells where delta > 0 |
| Negative delta | `--pos-danger` | `oklch(0.7 0.22 25)` — red | Stock movement delta cells where delta < 0 |
| Border | `--border` | `oklch(1 0 0 / 10%)` | Table borders, dialog borders, input borders |
| Muted foreground | `--muted-foreground` | `oklch(0.708 0 0)` | Helper text, timestamps, column headers |

**Accent reserved for:**
1. "Add ingredient" primary CTA button (`<Button>` default variant)
2. "Import CSV" secondary CTA button (`<Button variant="outline">`)
3. "Save" button inside IngredientForm Dialog
4. "Confirm Import" button inside CsvImportSheet

Accent is NOT used for: row hover states, table headers, badges, ledger reason chips.

**Low stock indicator:** when `quantity_on_hand <= reorder_point` and `reorder_point > 0`, apply `getRowClassName` returning `"bg-pos-danger/10"` on that DataTable row. This matches the `getRowClassName` prop already on `DataTable`.

Location: `src/widgets/IngredientsTable/index.tsx`

Uses `DataTable` from `shared/ui/DataTable.tsx` with the following column definition:

| Column key | Header | Cell content | Sortable | Width |

### CsvImportSheet (feature UI — Sheet)

Location: `src/features/import-ingredients-csv/ui/CsvImportSheet.tsx`

Container: `<Sheet>` — `side="right"` with `className="w-full sm:max-w-xl"`. Uses existing `sheet.tsx` from `shared/ui`.

UX flow (3 states):

**State 1 — File selection:**
```
SheetHeader
  SheetTitle: "Import ingredients"
  SheetDescription: {description copy}
SheetContent
  [Template download link — <a> text link with Download icon]
  <Label>CSV file</Label>
  <Input type="file" accept=".csv" />
  [Hint text: "UTF-8 encoded · max 500 rows"]
SheetFooter
  <Button variant="outline">Cancel</Button>
  <Button disabled>Import ingredients</Button>  {disabled until file selected}
```

**State 2 — Staged (file parsed, awaiting confirm):**
```
SheetContent
  <p className="font-semibold">Preview — {N} rows found</p>
  [Validation summary copy]
  {M > 0 && <FailedRowsList /> — ul.divide-y.rounded-md.border, max-h-48 overflow-y-auto}
SheetFooter
  <Button variant="outline" onClick={resetToStep1}>Choose different file</Button>
  <Button disabled={validCount === 0} onClick={handleConfirm}>Import {N} ingredients</Button>
```

**State 3 — Importing:**
```
SheetFooter buttons both disabled; confirm shows "Importing…" with Loader2 spinner
```

After success: Sheet closes. Toast shown. IngredientsTable query invalidated.

File parsing: client-side with `FileReader` + manual CSV split (no papaparse — no new deps). Validate each row with `IngredientSchema.safeParse()` from domain.ts.

### StockMovementsList (widget)

Location: `src/widgets/StockMovementsList/index.tsx`

Renders as a collapsible section below the IngredientForm (or as a separate panel when navigating to a single ingredient detail). For Phase 3, it is rendered inside the IngredientsTable row expansion or inside the edit Dialog as a scrollable section below the form.

**Rendering context:** Inside IngredientForm Dialog, below a `<div className="border-t pt-4">` divider (matches ComboAvailabilityEditor pattern in edit Dialog).

Uses `DataTable` with the following columns:

| Column key | Header | Cell content | Sortable |
|-----------|--------|--------------|---------|
| `created_at` | Date | `{date} {time}` formatted as `dd MMM yyyy HH:mm` | Yes (default: newest first) |
| `delta` | Change | `+{qty} {uom}` (text-pos-accent) or `−{qty} {uom}` (text-pos-danger); `font-mono text-sm` | No |
| `reason` | Reason | Capitalized: `Waste`, `Delivery`, `Correction`, `Physical count`, `Sale`, `Void`, `Refund` | No |
| `ref_type` | Source | `Manual`, `Order`, `Refund`, `Prep` — derived from `ref_type` enum | No |
| `ref_id` | Ref | First 8 chars of UUID + `…` in `font-mono text-xs text-muted-foreground`; `—` if null | No |

Props:
- `enableSorting={true}`, `initialSorting={[{ id: 'created_at', desc: true }]}`
- `searchable={false}` — ledger is read-only; no search
- No `onRowClick` — read-only, no row action
- No pagination limit (show all movements for the ingredient); parent scrolls
- `emptyState`: `<EmptyState icon={History} title="No movements recorded" description="..." />`

**No "Record adjustment" button inside StockMovementsList** — that button lives in the IngredientForm header area. This widget is purely read-only display.

## Interaction States

All interactive states must be implemented for each component:

| State | How to Handle |
|-------|--------------|
| Loading (initial) | `DataTable isLoading={true}` — shows `TableRowSkeleton` (5 rows) |
| Empty | `DataTable emptyState={<EmptyState>}` with action button |
| Error (query) | `<p className="text-sm text-destructive">Could not load…</p>` |
| Submitting (form) | Submit button disabled + "Saving…" text; no spinner required (matches CategoryForm) |
| Submitting (delete) | ConfirmDialog `isLoading={true}` — shows Loader2 spinner |
| Low stock | `getRowClassName` returns `"bg-pos-danger/10"` on affected rows |
| Positive delta | `text-pos-accent font-mono text-sm` in StockMovementsList delta cell |
| Negative delta | `text-pos-danger font-mono text-sm` in StockMovementsList delta cell |

## Accessibility

Inherits from locked decisions in `04-navigation-ui-flows.md`:
- All interactive elements reachable via keyboard
- Focus traps in Dialog and Sheet (shadcn defaults — do not override)
- Color contrast ≥ 4.5:1 on all text (met by CSS variable system)
- `aria-label` on all icon-only buttons (Pencil, Trash2)
- `role="status"` + `aria-live="polite"` on EmptyState (already in shared component)
- DataTable: `data-state` on selected rows (shadcn default)
