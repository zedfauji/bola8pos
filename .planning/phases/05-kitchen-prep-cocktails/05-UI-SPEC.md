## Visual Hierarchy

**KitchenPrepPage:** `BackToHomeButton` at top → `PageContainer` title "Kitchen Prep" → two stacked sections: (1) "Prep on hand" grid (2-column responsive grid of `PrepOnHandCard`s) and (2) "Recent batches" `DataTable`. The "New batch" `Button` lives in the `PageContainer` `actions` slot (top right, `variant="default"` — primary).

**PrepProductionForm Dialog:** Compact `max-w-sm` Dialog. Ingredient picker (IngredientAutocomplete filtered to `is_prep=true`) is the first and dominant element. Qty input below it. Notes textarea below qty (optional). `PrepBatchPreview` panel appears below Notes only when a prep ingredient with an associated recipe is selected. "Record batch" CTA is the sole primary button in Dialog footer.

**PrepBatchPreview panel:** `rounded-lg bg-muted border p-4` panel showing a table of raw ingredients to be consumed. Rows with insufficient stock are highlighted `text-pos-danger`. This is read-only, purely visual; no controls.

**HomeDashboard Kitchen Prep tile:** Same card pattern as existing tiles (KDS, Inventory). Visible to manager+ (and kitchen role with `produce_prep_batch` action). `ChefHat` icon from lucide-react.

**Structure:**
```
Card (rounded-lg border p-4 flex flex-col gap-2)

    span.text-sm.text-muted-foreground  — UOM

  div.h-1.rounded-full.bg-muted.overflow-hidden[aria-label="Stock level"]
```

**Props:**
```typescript
interface PrepOnHandCardProps {
  name: string;
  uom: string;
  qtyOnHand: number;
  reorderPoint: number | null;
}
```

**States:**
- Healthy (above reorder): `border-border`, qty in `text-pos-accent`
- Low (at or below reorder): `border-pos-danger`, qty in `text-pos-danger`
- Out (qty = 0): `border-pos-danger bg-pos-danger/5`, qty in `text-pos-danger font-semibold`

**Storybook story variants:** `Healthy` (qty above reorder), `Low` (at reorder), `OutOfStock` (qty=0), `NoReorderPoint` (reorder_point null — no bar shown).

Dialog form for recording a prep batch.

```
    [Notes]
    FormField label="Notes" (optional)
      <Textarea
        placeholder="e.g. Morning batch — double yield"
        value={notes}
        onChange={e => setNotes(e.target.value)}
```

#### `PrepBatchPreview` (`src/features/produce-prep-batch/ui/PrepBatchPreview.tsx`)

Read-only raw-consumption preview panel. Purely client-side calculation — no RPC call.

**Structure:**
```
div.rounded-lg.bg-muted.border.p-4.space-y-3 (aria-label="Ingredient consumption preview")
  [Header]
  p.text-sm.font-semibold "Raw material consumption preview"

  [Table]
  div.space-y-1
    [Header row]
    div.flex.items-center.justify-between.text-xs.text-muted-foreground.pb-1.border-b
      span "Ingredient"
      span "Quantity"

    [Data rows — one per recipe_item]
    {recipeItems.map(item => (
      div.flex.items-center.justify-between.py-1.text-sm (key=item.ingredientId)
        span.truncate.max-w-[60%]  — item ingredient name
          -- class: text-pos-danger if currentQtyOnHand < neededQty
        span.font-mono.tabular-nums  — "−{neededQty} {uom}"
          -- class: text-pos-danger if currentQtyOnHand < neededQty
    ))}

  [Credit row — the prep ingredient itself]
  div.flex.items-center.justify-between.pt-2.border-t.text-sm
    span.font-medium  — "{prepIngredient.name}"
    span.font-mono.tabular-nums.text-pos-accent  — "+{qtyProduced} {uom}"
```

**Computation:** `neededQty = qtyProduced × item.qty / recipe.yieldQty` (pure function `computePrepConsumption` exported from `@shared/lib/prep-math`).

**Props:**
```typescript
interface PrepBatchPreviewProps {
  recipe: RecipeWithItems;       // fetched by parent feature; null guard done above render
  qtyProduced: number;
  prepIngredient: { name: string; uom: string };
  currentQtyOnHand: Map<string, { qty: number; uom: string }>;  // for insufficient-stock highlighting
}
```

**Insufficient stock highlighting:** Any row where `currentQtyOnHand.get(ingredientId).qty < neededQty` renders ingredient name and qty in `text-pos-danger`. No blocking — preview is informational. The trigger itself will raise `INVENTORY_NEGATIVE` if stock is actually insufficient at insert time.

## Interaction Contracts

### PrepProductionForm — state machine

```
CLOSED
  → [New batch button tap] → OPEN (all fields empty)

OPEN
  Substates: EMPTY, INGREDIENT_SELECTED, RECIPE_LOADED, SUBMITTING

  EMPTY
    → select prep ingredient → INGREDIENT_SELECTED
      (fetch recipe for this ingredient in background)

  INGREDIENT_SELECTED
    Substates: LOADING_RECIPE, RECIPE_FOUND, NO_RECIPE
    → recipe fetch resolves → RECIPE_FOUND (PrepBatchPreview appears) or NO_RECIPE (no preview)
    → enter qty (> 0) → form becomes valid; Confirm enabled

  RECIPE_FOUND or NO_RECIPE
    → isValid = selectedIngredientId !== null && parsedQty > 0
    → "Record batch" tap (isValid=true) → SUBMITTING

  SUBMITTING
    → Both buttons disabled; "Record batch" shows LoadingSpinner
    → [Success] → CLOSED + success toast
    → [Error PREP_INGREDIENT_REQUIRED | INVENTORY_NEGATIVE | other] → OPEN + error toast
      (form state preserved; user can adjust qty or ingredient)
```

### Ingredient selection — is_prep filter

`IngredientAutocomplete` in `PrepProductionForm` receives only `is_prep=true` ingredients (passed as `ingredients` prop from the parent widget which calls `useIngredientsActive()` and filters client-side). The autocomplete input is a Command-based popover (same pattern as `IngredientAutocomplete` from Phase 4). Searching by name only. No category grouping required in this phase.

### PrepOnHandCard — no interaction

Cards are purely read-only. No click handler. No hover effect beyond `border-pos-danger` color change for low-stock. Tapping does nothing.

### IngredientsTable filter — no new interaction

The `filterPrep` prop is controlled by the parent widget. No UI toggle is shipped in this phase — the prop exists for the `KitchenPrepDashboard` to call with `filterPrep="prep"` and for the main Settings Ingredients tab to call with `filterPrep="all"` (unchanged). No filter button is rendered inside `IngredientsTable`.

### KitchenPrepRoute guard

Mirrors `KdsRoute`. Checks `can('produce_prep_batch')`. Redirects to `/home` if not authorized. Kitchen, manager, and admin roles pass.

## Accessibility Contract

| Requirement | Implementation |
|------------|----------------|
| All interactive elements keyboard-reachable | Dialog, Button, Input, Textarea, Command popover use shadcn/Radix defaults — fully keyboard navigable |
