## Spacing Scale

All values are multiples of 4. Source: 8-point grid, Tailwind default scale. No custom spacing tokens are defined — use Tailwind utility classes directly.

| Token | Value | Tailwind Class | Usage in This Phase |
|-------|-------|---------------|---------------------|
| xs | 4px | `p-1`, `gap-1` | Icon gaps inside table cells, badge padding |
| sm | 8px | `p-2`, `gap-2` | Compact table cell padding, filter row gap |
| md | 16px | `p-4`, `gap-4` | Default widget internal padding, filter section spacing |
| lg | 24px | `p-6`, `gap-6` | ReportsPage `main` horizontal padding (existing: `p-6 md:p-8`) |
| xl | 32px | `p-8` | ReportsPage desktop padding (existing) |
| 2xl | 48px | — | Not used in this phase |
| 3xl | 64px | — | Not used in this phase |

**Exceptions:**
- Touch targets on tablet-facing screens (WaitlistQueue, SeatPartySheet trigger buttons): minimum 44×44px. Enforce with `min-h-[44px] min-w-[44px]` where current `size="sm"` buttons fall below threshold. Source: S6-12 paper-cut spec.
- `space-y-4` (16px vertical rhythm) is the standard between widget sub-sections — matches existing `CategoryRevenuePanel` and `HourlyBreakdownPanel` patterns.

## Typography

All sizes use the Geist Variable font. No new font sizes are introduced in Phase 8 — match the existing 4-step scale already in use.

| Role | Size | Weight | Line Height | Tailwind Class | Usage in This Phase |
|------|------|--------|-------------|----------------|---------------------|
| Body | 14px | 400 (regular) | 1.5 | `text-sm` | Table cell content, filter labels, description text |
| Label | 14px | 500 (medium) | 1.4 | `text-sm font-medium` | Table column headers (`TableHead`), metric card labels |
| Heading | 16px | 600 (semibold) | 1.3 | `text-base font-semibold` | Report widget section headings, metric callout values |
| Display | 20px | 600 (semibold) | 1.2 | `text-xl font-semibold` | Large metric numbers in WaitlistAnalyticsReport metric cards |

**Weights used:** regular (400) + semibold (600). Medium (500) used only for `TableHead` labels (matches existing panels). This is a maximum of 3 effective weights — within the 2-weight guidance; the medium is reserved for headers only.

**Monospace exception:** Tabular-numeric cells use `tabular-nums` utility (not a new font-family). Timestamp cells in RefundsRegister use `font-mono` for alignment — matches `HourlyBreakdownPanel` precedent.

## Color

All colors are CSS variables resolved from `src/app/globals.css`. No hardcoded hex values. Dark mode handled automatically by `@media (prefers-color-scheme: dark)`.

| Role | CSS Variable (dark mode value) | Usage |
|------|-------------------------------|-------|
| Dominant (60%) | `--background` = `oklch(0.145 0 0)` | Page background, table background |
| Secondary (30%) | `--card` = `oklch(0.205 0 0)` | Metric callout cards, `bg-muted/30` banner rows |
| Accent (10%) | `--accent` = `oklch(0.269 0 0)` | See reserved list below |
| Destructive | `--destructive` = `oklch(0.704 0.191 22.216)` | Variance >±10% rows in RecipeVarianceReport only |

**Accent reserved for:**
- Active `TabsTrigger` indicator (existing shadcn behavior, no code change needed)
- Focus ring (`--ring`) on interactive elements

**Semantic chart colors (report-specific, not in 60/30/10 framework):**
These are additive to the color contract, not part of the accent 10%. Use these exact classes — do not introduce new CSS variables.

| Semantic | Class | Usage |
|----------|-------|-------|
| Positive / peak | `text-emerald-500`, `bg-emerald-500/5`, `border-l-emerald-500` | Highest-performing combo row highlight in ComboMixReport; matches HourlyBreakdownPanel precedent |
| Warning / slow | `text-amber-400`, `bg-amber-500/5`, `border-l-amber-400` | Lowest-performing rows where applicable; matches HourlyBreakdownPanel |
| Variance danger | `text-destructive` | RecipeVarianceReport rows where `|variance_pct| > 10`. Use shadcn semantic token, not Tailwind `text-red-*` |
| Muted metadata | `text-muted-foreground` | Reason column in RefundsRegister, override count when 0, all secondary descriptors |

**Chart color variables (`--chart-1` through `--chart-5`):**
Use for recharts stacked bar series in ComboMixReport. Declared in `globals.css` as neutral grays in both light and dark. For legibility in a stacked bar with multiple combos, override the first 4 series with these semantic classes applied to recharts `<Bar fill>` props:
- Series 1: `var(--chart-1)` = `oklch(0.87 0 0)`
- Series 2: `oklch(0.72 0.19 145)` (`--pos-accent`, green) — highest-revenue combo
- Series 3: `var(--chart-3)` = `oklch(0.439 0 0)`
- Series 4: `var(--chart-4)` = `oklch(0.371 0 0)`
- Series 5+: `var(--chart-5)` = `oklch(0.269 0 0)`, repeated with opacity steps

## Per-Widget Layout Contracts

### Pattern: Standard Report Widget

Every new report widget follows this exact DOM structure (verified from `CategoryRevenuePanel`):

```
<div className="space-y-4">
  <div className="flex justify-end">
    <ExportButtons reportType="..." data={...} />
  </div>
  {/* optional: metric callout banner */}
  <div className="rounded-md border">
    <Table>…</Table>
  </div>
</div>
```

Loading state: `if (isLoading) return <LoadingSpinner />;` — placed before any data access.

Empty state: `<EmptyState icon={IconName} title="..." description="..." />` — see Copywriting Contract for exact strings.

## Paper-Cut UI Contracts

### Touch Target Sizing

Applies to: WaitlistQueue action buttons, SeatPartySheet trigger, AddWaitlistEntryForm submit button, any `size="sm"` button in a tablet-facing page.

Rule: Every interactive element visible on `/waitlist`, `/pool-tables`, and `/pool-tables/:tableId` must meet 44×44px minimum.

Implementation: Apply `min-h-[44px] min-w-[44px]` to button wrapper or use `size="default"` instead of `size="sm"` on these screens. Do not change button styles on the ReportsPage (desktop-only, keyboard-primary).

### Focus Trap

Applies to: `SeatPartySheet`, `AddWaitlistEntryForm` (rendered inside Sheet), `RefundSheet`.

Rule: When a `Sheet` opens, focus must move to the first interactive element inside `SheetContent`. Verify `SheetContent` uses `onOpenAutoFocus={(e) => { e.preventDefault(); firstRef.current?.focus(); }}` pattern. Tab cycling must remain inside the Sheet until closed. `Escape` must close the Sheet.

Implementation: Check each Sheet component for explicit `initialFocus` or `onOpenAutoFocus` prop. If absent, add a `ref` to the first focusable element and wire it.

### Loading Skeleton Coverage

Every async widget must show a skeleton while `isLoading` is `true`. Current pattern uses `<LoadingSpinner />`. This is acceptable for report widgets (full-panel spinner). For metric cards in `WaitlistAnalyticsReport`, use `<Skeleton className="h-12 w-24" />` per card slot instead of the spinner.

Rule: No widget renders `null` or an empty `<div>` while loading. The loading state must be visually distinct from the empty state.

### Error Toast Copy Standardization

Current inconsistency: some toasts say "Something went wrong", others say feature-specific strings.

Rule for Phase 8 paper-cut sweep: All `toast.error()` calls in the 8 modified waitlist files (listed in RESEARCH.md Paper-Cut Inventory) must use:
- Mutation failure: `"Action failed. Try again."`
- Export failure: `"Export failed. Try again."`

Do not change toasts in other files unless they are directly touched by Phase 8 work.
